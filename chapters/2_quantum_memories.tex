\subsection{Pauli Group}\label{ssec:intro_pauli_group}

The Pauli group $\mathcal{P}_n$ is made up of tensor products of the $n$ one qubit Pauli group without phase $P \in \hat{\mathcal{P}}_1 = \{ I,X,Z,Y \}$ 
\begin{equation}
I =
\begin{pmatrix}
1 & 0 \\
0 & 1
\end{pmatrix}, \qquad
X =
\begin{pmatrix}
0 & 1 \\
1 & 0
\end{pmatrix}, \qquad
Y =
i
\begin{pmatrix}
0 & -1 \\
1 & 0
\end{pmatrix}, \qquad
Z =
\begin{pmatrix}
1 & 0 \\
0 & -1
\end{pmatrix}
\end{equation} 
together with an overall phase $\{\pm1,\pm i\}$
\begin{equation}
    \mathcal{P}_n = \{c P_1\otimes P_2 \otimes ... \otimes P_n | c \in \{\pm 1, \pm i\}; P_i \in \hat{\mathcal{P}}_1 \forall i \}
\end{equation}

% Make the 
Note that a global phase of a state has no observable consequence and as such we can omit it in most cases.
Following this argumentation it can be sometimes useful to omit the overall phase of the operator as well in some cases.
When we want to talk about the Pauli group without a phase we will refer to it as $\hat{\mathcal{P}}_n$.

At this point we can introduce the weight of an Pauli operator $P\in\mathcal{P}_n$ indicated by $|P|$ as the number of non identity Paulis in the tensor product that defines $P\in\mathcal{P}_n$.
This quantity will become interesting when we talk about the distance of codes and what errors are guaranteed to be correctable \ref{sssec:distance_stabilizer_formalism}. 

The Pauli group is closed under multiplication, especially $Y = i X Z$, as such it is a group.
Each Pauli squared, disregarding the phase, is identical to identity
\begin{equation}
    I = X^2 = Z^2 = Y^2.
\end{equation} 
Resulting from this any tensor product of Paulis, is also its own inverse, when we disregard the overall phase
\begin{equation}
    \tilde{P} \tilde{P} = I \forall \tilde{P} \in \hat{\mathcal{P}}_n.
    \label{eq:pauli_squared_is_identity}
\end{equation}
Taking the phase into account the square of an element of the Pauli group is $\pm I $, depending on the overall phase.  
This property is important when we want to find a recovery for a given Pauli error, see section \ref{ssec:stabilizer_summary_qec_cylce}. 
To recover the error $P$ we just reapply the same Pauli, and in the process we might change the overall phase but otherwise act with the identity.

Another property of the Pauli group is that all their elements either commute or anticommute.
For the one qubit Pauli group all non trivial pairings anticommute, we will use $\{A,B\}$ to indicate the anticommutator, 
\begin{equation}
    \{X,Z\} = \{X,Y\} = \{Z,Y\} = 0.
\end{equation} 
Elements of the $n$ qubit Pauli group $\mathcal{P}_n$ can also non trivially commute if they have an even number of anticommuting elements. 

Each Pauli operator splits the Hilbert space in half, with each half either being associated with the eigenvalue $+1$ or $-1$.
A fact that we use to define the codespace.

% Following can be written better!
We can use the $n$-qubit Pauli group as a basis to express any quantum channel. 
As it spans the space of all $n$-qubit linear operations.
Which will become important when we discuss general errors.

We will refer to $X$-type Pauli operator as bit-flip, as this error flips the $\ket{0}$ state to the $\ket{1}$ state and vice versa 
\begin{equation}
    X \ket{Psi} = X(\alpha \ket{0} + \beta \ket{1}) = \beta \ket{0} + \alpha \ket{1}.
\end{equation}
And we will refer to the $Z$ Pauli operator as phase-flip, as it maps the $\ket{1}$ basis state to $-\ket{1}$, while leaving the $\ket{0}$ basis state unchanged
\begin{equation}
    Z \ket{Psi} = Z(\alpha \ket{0} + \beta \ket{1}) = \alpha \ket{0} - \beta \ket{1}.
\end{equation}

The $Y$ Pauli is just bit- and phase-flip applied with an additional overall phase of $i$.

\subsection{Stabilizer Group}

The stabilizer formalism is based around a subgroup of the Pauli group $\mathcal{P}_n$ called the stabilizer group $\mathcal{S}$. 
The reason for naming the subgroup stabilizers becomes obvious in the next subsection \ref{ssec:codespace}, when we talk about the codespace.

By choosing the stabilizer group one defines the specific stabilizer code and the specific encoding of the QEC code.
We will later in section \ref{sec:kitaev_surface_code} discuss a specific choice but here we take a general view upon stabilizer codes. 
One can freely choose which operators to use to define a stabilizer group with the following limitations for the resulting stabilizer group.
It must be an Abelian subgroup of the $n$-qubit Pauli group $\mathcal{P}_n$ and it is not allowed to contain $-I$. 

Many of the following definitions will be formulated by only taking a set of $m$ linear independent generators of the stabilizer group, which we will label as $S_i$, such that the stabilizer group can be written as $\langle S_1, S_2, ..., S_m\rangle= \mathcal{S}$.
To shorten the notation even further we will refer to the set of linear independent generators of the stabilizer group by $\mathcal{S}_G$.

We will also refer to the linear independent generators of the stabilizer group by generators to shorten the following text.


\subsection{Codespace}\label{ssec:codespace}% rework this headline 

As described earlier the basic idea behind QEC codes is to encode the logical information of $k$ logical qubits into $n$ physical qubits.  
The space in which the logical states lay is called the codespace $\mathcal{L}$, which is a subspace of the full $n$-qubit Hilbert space $\mathcal{H}_{2^n}$. 

Using the stabilizer group defined in the previous subsection, we can define the codespace $\mathcal{L}$ as being the space spanned by all states $\ket{\Psi}$ which are invariant under all elements of the stabilizer group.

\begin{equation}
    \mathcal{L} = \left\{  \ket{\Psi} \in  \mathcal{H}_{2^n} : S \ket{\Psi} = \ket{\Psi} \forall S \in \mathcal{S} \right\}
    \label{eq:defintion_codespace_whole_group}
\end{equation}

Given that each element of the stabilizer can be generated by the set of generators $\mathcal{S}_G$, we can rewrite this definition as 

\begin{equation}
    \mathcal{L} = \left\{  \ket{\Psi} \in  \mathcal{H}_{2^n} : S \ket{\Psi} = \ket{\Psi} \forall S_i \in \mathcal{S}_G \right\}.
    \label{eq:defintion_codespace_generators}
\end{equation}

Given that we have $n$ physical qubits into which we encode the logical information and given we use a stabilizer group with $m$ linear independent generators, we can encode $k=n-m$ logical qubits into the resulting codespace. 
Which we can understand by following the argumentation:
Each of the $m$ linear independent generators splits splits the $2^n$ dimensional Hilbert space into two subspaces associated with the eigenvalues $+1$ and $-1$.
The resulting codespace is therefore a $2^k=2^n/2^m$ dimensional Hilbert space.
This subspace is characterized by being the $+1$ eigenspace of all generators (and all stabilizers).

We will refer to the states which are in the codespace, as codewords $\ket{\Phi}\in \mathcal{L}$.

Here we note explicitly that acting with an element of the stabilizer group on any codeword is equivalent to act with the identity on them,
given that the codewords states are defined as being $+1$ eigenstates of all elements of the stabilizer group
\begin{equation}
    S \ket{\Phi} = \ket{\Phi}.
    \label{eq:stabilizer_equivalence}
\end{equation}
We will refer to this intrinsic property of stabilizer codes by calling it the stabilizer equivalence.



\subsection{Syndrome}\label{ssec:syndrome}

Now that we have defined the codespace, we can detect if a given eigenstate of the stabilizer group\footnote{
    Only such states are of interest, because they are the basis of our logical information. % Improve why only those states are relevant 
} is in the codespace. 
We determine the eigenvalues of the generators of the stabilizer groups for the given state $\ket{\Psi}$.
Based on the eigenvalues we can assign the state a $m$-bit string, the syndrome $\sigma(\ket{\Psi}) \in \mathbb{Z}_2$, following 
\begin{equation}
    S_i \ket{\Psi} = (-1)^{\sigma_i} \ket{\Psi},
    \label{eq:state_syndrome}
\end{equation}
If the state is a $+1$ ($-1$) eigenstate of $i$-th generator, the $i$-th bit of the syndrome takes the value $0$ ($1$).
By definition the state is only in the codespace if it is in the $+1$ eigenstate of all generators, which is equal to the syndrome being all $0$.
We will refer to the syndrome with all bits set to $0$ as the trivial syndrome.
All states with different syndromes lie outside of our defined codespace. 

Note that given we have $m$ generators, there are $2^m$ possible different syndromes.

We can construct a similar syndrome string for elements of the Pauli group $E\in \mathcal{P}_n$. 
Note that we assumed that we can represent errors as elements $E\in \mathcal{P}_N$. 
To define the error syndrome $\sigma(E) \in \mathbb{Z}_2$ we first apply the operator $E$ to any codeword $\ket{\Phi}$.
Then we determine the syndrome of the resulting state following the procedure above.
We define the resulting syndrome as the error syndrome or the operator $E$
\begin{equation}
    \sigma(E) = \sigma(E\ket{\Phi}); E \in \mathcal{P}_n; \Phi \in \mathcal{L}.
    \label{eq:error_syndrome}
\end{equation}
The resulting syndrome tells us how the operator $E$ would change the syndrome of a codeword state.

Looking at the definition of the $i$-th bit of the error syndrome, 
\begin{equation}
    S_i E \ket{\Phi} = (-1)^{\sigma_i} E S_i \ket{\Phi}.
\end{equation}
we can identify that the $i$-th bit of the error syndrome is defined by the commutation relation between the error and the $i$-th generator.
It is independent of the choice of the codeword.
If the generator and the error operator commute (anticommute) the error syndrome of the $i$-th bit has the value $0$ ($1$).
Given that both the error operator and the generator are Pauli operators, they either commute or anticommute.

Note that the choice of the specific codeword does not matter to determine the syndrome of an error operator.
The error operators changes the syndrome of each codeword by its associated error syndrome. 
% Are some explanations missing ?
Therefore it maps the whole codespace onto a different subspace where the eigenvalue for all stabilizers is no longer $+1$ but the syndrome of the error operator for each codeword.
This is a key result.
% Should I refer to Knill Laflamme EC condition?

It tells us that all of those $2^m$ subspaces are isomorphic to the codespace, a fact that we are gonna use in subsection \ref{ssec:cosets_of_the_centralizer}.

Note that we can reinterpret any of those subspaces, e.g. the subspace with the syndrome $\tilde{\sigma}$, to be the codespace by redefining the stabilizer group.
For this we redefine the $i$-th generator $S_i$ to be $-S_i$ for each bit in the syndrome $\tilde{\sigma}$ which take the value $1=\tilde{\sigma}$. 
When we use the resulting set of linear independent generators to define the stabilizer group, the subspace corresponding to the syndrome $\tilde{\sigma}$ using the previous set on linear independent generators, 
has now trivial syndrome assigned to it and is the codespace. % this sentence is to long and cumbersome!
We will use this fact in the subsection about Pauli Frame Tracking again \textcolor{red}{Put ref here}.

Concluding we have $2^m$ subspaces which are isomorphic to the codespace, one for each syndrome. 
All of those subspaces are orthogonal to each other, because they have different eigenvalues/different syndromes.
Each of those subspaces is $2^k$  dimensional to encode the $k$-logical qubits, where $k=n-m$.
Therefore we can decompose the full Hilbert space $\mathcal{H}_{2^n}$ into a direct sum of the $2^m=2^{n-k}$ subspaces, each one associated with a different syndrome.

\subsubsection{Distance}\label{sssec:distance_stabilizer_formalism}
% Distance
At this point we can introduce an characteristic quantity of QEC codes. % more general of QEC codes
The distance $d$ which represent the minium weight\footnote{The weight of an operator is the number of qubits on which the operator acts non trivially.} that of a non identity logical operator/undetectable error.
\begin{equation}
    d = \min_{E\in \mathcal{C}(\mathcal{S})\setminus \mathcal{S}} |E|
\end{equation}
A distance $d$ code can correct $t$ errors, with 
\begin{equation}
    t =  \left \lfloor \frac{d-1}{2} \right \rfloor.
    \label{eq:correctable_errors}
\end{equation}
The equation above is easier if we follow the chain of reasoning presented by Terhal in \cite{terhal_qec_formalism}.

If we have a weight $t$ error $E$, our decoder can find an correction $R$ with the same syndrome, at at most weight $t$.
The combined operators $RE$ has then at most weight $2t$, which by the definition above smaller than the distance $d$.
This combined operators has no error syndrome, therefore belongs to the centralizer and can by definition of the distance not be a non identity logical operator.
Therefore it is a stabilizer and acting with a stabilizer on a codeword does not change it and therefore the we can recover this error. 

And now we have all necessary quantities to introduce a notation characterizing stabilizer codes.
We refer to stabilizer codes by their set of characteristic number as $[[n,k,d]]$ codes.
$n$ being the number of physical qubits, $k$ being the number of encoded qubits and $d$ is the distance of the code.

In subsection \ref{ssec:intro_centralizer} we introduced the centralizer. 
There we discovered that the centralizer contains the realizations of the logical operators on the encoded qubits (and the stabilizer group,

In this subsection we will show the underlying group theoretic structure of the centralizer, following the work of Gottesmann in \cite{gottesmann_book} (Proposition 3.10).

Central to this argumentation is that if and only if two operators from the centralizer $L_1, L_2 \in \mathcal{C}(\mathcal{S})$ act the same way on each codeword $\ket{\Phi}$, 
implying that they realize the same logical operation, 
they are in the same cosets of the stabilizer group.

To proof that we start with our assumption about the two operators, they are acting identical on all codeword states
\begin{equation}
    L_1 \ket{\Phi}=L_2 \ket{\Phi}; L_1, L_2 \in \mathcal{C}(\mathcal{S}); \forall \ket{\Phi} \in \mathcal{L}.
\end{equation}
By defining $M =  L_1^\dag L_2$, we can rearrange this to
\begin{equation}
    M \ket{\Phi} = +1\ket{\Phi}  ; \forall \ket{\Phi}\in \mathcal{L}.
\end{equation}

We can see that the operator $M$ is by definition (see eq. \ref{eq:defintion_codespace_whole_group}) an element of the stabilizer group, because all codewords are in its $+1$ eigenstates.

If we rearrange the definition of $M$ we get the that $L_1$ and $L_2$ are in the same coset of the stabilizer group
\begin{equation}
    L_2 = L_1 M \text{ with } M \in \mathcal{S}. 
\end{equation}

Thus we can conclude that all the different realizations of the same logical operator are contained in the same coset of the stabilizer group.


All elements of the same coset of the stabilizer group act therefore the same on all codewords.
They all realize the same logical operation.

    Note that we keep track of the phase. One can do the same analysis for the sets in which we disregard the global phases.
} and is isomorphic to the Pauli group $\mathcal{P}_k$.
We can write this as the quotient group
\begin{equation}
    \mathcal{C}(\mathcal{S})/\mathcal{S} \cong \mathcal{P}_k.
\end{equation}
We can therefore realize logical operations on the encoded state by using an operator of the corresponding\footnote{

coset of the stabilizer group in the centralizer.
The exact choice of element from the coset is not important, all elements are logical equivalent and they only differ by an element of the stabilizer group, 
which by stabilizer equivalence (eq. \ref{eq:stabilizer_equivalence}) is to say, they do not differ.

\subsubsection{Cosets of the Centralizer}\label{ssec:cosets_of_the_centralizer}

In the previous subsection we discovered that the centralizer, which can be described as the set of all Pauli operators which have the trivial error syndrome, can be partitioned into cosets of the stabilizer group. 
Each of those cosets is associated with a logical operator on the codespace.

In subsection \ref{ssec:syndrome} we discovered that each syndrome is associated with a subspace and each of those subspaces is isomorphic to the codespace.
We also showed that we can change the syndrome of any associated space by redefining the stabilizer group.
Following the same idea we can look at all the elements of the Pauli group $\mathcal{P}_n$ and group them into sets defined by their error syndrome.

The resulting structure is that all the operators with the same error syndromes are collected in cosets of the centralizer, which therefore can be associated with this syndrome.
Similar to the section above where all operators associated to the trivial syndrome are collected in normalizer.
% Should I show a proof for that/ refer to gottesmann for the proof?
We can choose for each syndrome a representative error $E_{\sigma}$.
In the following we assume that we fix this representative error once for all syndromes.

The whole Pauli group $\mathcal{P}_n$ can then be partitioned into cosets of the centralizer of the stabilizer group, where we go over all possible $2^m$ syndromes
\begin{equation}
    \mathcal{P}_n = \bigcup_{\sigma \in \{0,1\}^m} E_{\sigma} \mathcal{C}(\mathcal{S}).
\end{equation}

All of those $2^m$ cosets of the centralizer have a same inner structure.
Each of those cosets of centralizer can be partitioned into the cosets of the stabilizer group associated with a logical operator, as described in equation \ref{eq:centralizer_partitioned_into_cosets_of_stabilizer_group},
where each of those cosets also inherits the representative error $E_\sigma$ of the associated syndrome.

As a result we can partition the whole Pauli group into cosets of the stabilizer group
\begin{equation}
    \mathcal{P}_n = \bigcup_{\sigma \in \{0,1\}^m} \bigcup_{\bar{P} \in\bar{\mathcal{P}}} E_{\sigma} \bar{P} \mathcal{S}.
\end{equation}
The resulting structure is illustrated in figure \ref{fig:stabilizer_formalism_cosets_of_centralizer}.

As such if we fix, the stabilizer group $\mathcal{S}$, the set of realizations of the logical operators $\bar{\mathcal{P}}$ and the canonical representation error associated with each syndrome $E_\sigma$,
we can rewrite any operator from the Pauli group $P \in \mathcal{P}_n$ as a combinations of those 
\begin{equation}
    P =  E_\sigma \bar{L} S \forall P \in \mathcal{P}_n; S \in \mathcal{S}, \bar{L} \in \bar{\mathcal{P}}.
    \label{eq:pauli_rewriting_stabilizer_formalism}
\end{equation}
Note that here the overall phase is contained in the logical operator $\bar{L}$.
If we do not care about the phase we can just use the realizations of logical operators without phases.

\subsection{Summary: A QEC Cycle using stabilizer formalism}\label{ssec:stabilizer_summary_qec_cylce}

% rewrite this whole section 

We discussed some details about the formalism of the stabilizer codes and the underlying structure, but how does this help us to detect and correct errors?
To see why the previous analyzed structure is useful, we will talk through one QEC cycle, which describes the process of detecting and correcting an error.

To be able to detect errors we start in a logical codeword state $\ket{\Phi}$.
We will talk at a later point about how to initialize such a state. \textcolor{red}{put a ref here}

Note that we do not care which codeword state this is, as such we do not know the logical information encoded in those $k$ qubits.
And note that the logical operators, by definition, all commute with the stabilizers and therefore we can construct the syndrome, without collapsing any possible superposition of codeword states or learning anything about the encoded state. 

If an error operator\footnote{
    We still assume that our error operator is an element of the Pauli group $\mathcal{P}_n$.
} $\tilde{E}$ acts on our codeword state $\ket{\Phi}$. 
We get the resulting faulty state $\tilde{E}\ket{\Phi}$.

Our goal at the end of the cycle is to recover the original codeword state $\ket{\Phi}$.
To achieve this we need to find a way to inverse the error $\tilde{E}$.
As discussed in section \ref{ssec:intro_pauli_group}, the square of any operator is the identity, in addition to some global phase which has no physical relevance\footnote{As the phase is not relevant we will ignore is for the sake of clarity.}. 
Therefore we can recover our original codeword state if we apply the exact same error to our faulty state $\tilde{E}\ket{\Phi}$.
As such our goal during the QEC cycle is to identify the error operator $\tilde{E}$.
We can then apply the same error operator again to recover our original codeword state $\tilde{E}\tilde{E}\ket{\Phi} = \ket{\Phi}$.

We start the process of identifying the occurred error by measuring the syndrome of this state, which will give us the error syndrome of the error operator $\sigma(\tilde{E})$. 
The exact circuit/approach of extracting the syndrome of a given state will be discussed in section \textcolor{red}{Put ref here}.
The extracted error syndrome points us directly to the coset of the centralizer which contains the occurred error $\tilde{E} \in E_{\sigma(\tilde{E})}\mathcal{C}(\mathcal{S})$.
We went into great detail about this connection in the previous section \ref{ssec:cosets_of_the_centralizer}.

We thus know that the error $\tilde{E}$ must be an element of the set $\tilde{E}\in E_{\sigma(\tilde{E})}\mathcal{C}(\mathcal{S})$.
We do not know yet from which coset of the stabilizer group in the cosets of the centralizer this error is from.

Each coset of the stabilizer in the centralizer realizes a logical operator on the encoded qubit.
Asking the question, which coset the error belong to is asking the question, which logical error $\bar{L}_{\tilde{E}} \in \bar{\mathcal{P}}$ does the error contain.
Here we adapt the rewriting introduced in equation \ref{eq:pauli_rewriting_stabilizer_formalism} to the occurred error 
\begin{equation}
    \tilde{E} = E_{\sigma(\tilde{E})}\bar{L}_{\tilde{E}}\tilde{S}.
\end{equation}

The problem of determining from which cosets of the stabilizer the error is from, or which logical operator it includes, is the so called decoding problem.
It is handled by so called decoders. % is solved here the correct world
We will go into more detail about possible approaches to decode a syndrome in the following section \ref{sec:decoding_problem_general}.
For now lets assume our decoder give us a prediction from which coset of the stabilizer group the error operator should be from.
This prediction can be represented as a logical operator, which we refer to as $\bar{L}_D\in\bar{\mathcal{P}}$.

We can then construct a possible recovery operation $R$ from the given parts as,
\begin{equation}
    R = E_{\sigma(\tilde{E})}\bar{L}_{D} S_R.
\end{equation}
To reiterate we got the representative error $E_{\sigma(\tilde{E})}$ by the syndrome of the error.
The logical component $\bar{L}_D \in \bar{\mathcal{P}}$ is predicted by the decoder.
Note that we do not have any information about the stabilizer component $\tilde{S}$ of the occurred error, but as we will see later we can choose any stabilizer element $S_R$ for our recovery.

By applying this recovery operation $R$, we hope to be able to undo the previously occurred error $E$, so we would succeed if  
\begin{equation}
    R E \ket{\Phi} = \ket{\Phi}.
\end{equation}

The final question is did we recover the state correctly?
So lets look at the state\footnote{
    Given that we ignore the phases we could also apply $R\tilde{E}$, instead of $R^\dag\tilde{E}$ and we would change the phase by commuting the operators to the correct position.
    But I found it more illustrative to recover the Pauli by applying the adjoint.
    } $R^\dag \tilde{E} \ket{\Phi}$,
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = S_R^\dag \bar{L}_D^\dag E_{\sigma(\tilde{E})}^\dag E_{\sigma(\tilde{E})}\bar{L}_{\tilde{E}}\tilde{S}\ket{\Phi}.
\end{equation}
Given that we ignore the phase all pauli operators are hermitian, we can ignore the daggers.
Applying any Pauli operator twice is identical to the identity, so $E_{\sigma(\tilde{E})}^\dag E_{\sigma(\tilde{E})} = I$, which results in 
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = S_R \bar{L}_D \bar{L}_{\tilde{E}} \tilde{S}\ket{\Phi}.
\end{equation}
Here we follow the assumption that we fix the representative errors in the beginning.
The stabilizer group element $S_R$ commutes with the realizations of the logical operators.
They all belong to the centralizer, as we have established.
Therefore we can rewrite the expression as 
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = \bar{L}_D \bar{L}_{\tilde{E}} S_R \tilde{S}\ket{\Phi}.
\end{equation}
By the concept of stabilizer equivalence, or just by the definition of the codeword states, the stabilizers acting on the codeword states do not change it and therefore can be absorbed into it.
Resulting in
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = \bar{L}_D \bar{L}_{\tilde{E}} \ket{\Phi}.
\end{equation}

For this recovery to be successful we thus just have the condition $\bar{L}_D=\bar{L}_{\tilde{E}}$. 
In this case we would recover our original codeword state $\ket{\Phi}$.
Note that the choice of stabilizer element of the recovery $S_R$ does not matter, as such we only need to determine the correct cosets of the stabilizer group, not the identical error operator.

If they are not identical, they would implement a different logical operator $\bar{L}_\Delta = \bar{L}_D \bar{L}_{\tilde{E}}$, 
as indicated by the isomorphism between the chosen realizations of the logical operators $\bar{\mathcal{P}}$ and the Pauli group for the the encoded logical qubit $\mathcal{P}_k$.
Which then would change our codeword resulting in an unsuccessful recovery.

We thus see that the crucial part of the QEC cycle and its success rate is dependent on solving the decoding problem correctly.
As such the choice of decoding strategy is quite influential on our QEC code performance and 

The success rate will be later quantified as the logical error rate $p_L$, the rate at which we make logical mistakes in our recovery.

\section{Decoding Problem}\label{sec:decoding_problem_general}

The decoding problem is the task of determining the occurred error based on the observed syndrome.

To shortly recap, given the error syndrome we can identify the cosets of the centralizer to which the error belongs.
For a successful recovery we also need to know to which coset of the stabilizer group, disregarding the phase, the error belongs to.
The coset of the stabilizer group defines determined logical component of the occurred error.
The syndrome does not provide any information\footnote{
    Knowing the logical state before or after the error occurred would lead to a wavefunction collapse/a projection onto the measured state and as such would destroy possible superpositions of the logical state. 
    We are therefore limited in the information we can extract from the state. See \cite{knill_laflamme}.
} about the logical state of the codeword or the logical operator applied. 
The decoder therefore needs to use a different approach do predict which logical component might have occurred.

For this we need as an additional resource the error model.
This error model assigns a probability to each possible error operator $\text{Pr}(E) \forall\hat{\mathcal{P}}_n$.
We will discuss the exact error models used in this thesis in section \textcolor{red}{Put ref here}.

We will use two different decoding strategies in this thesis.
The first is called Maximum Likelihood Decoding (MLD), as it is the decoding approach which the highest likelihood of returning a successful recovery.
This approach can be summarized in the context of stabilizer codes as finding the coset of the stabilizer group, in the coset of the centralizer with the corresponding syndrome, with the highest probability.
As such we take into account all possible errors.
In return we get the optimal possible success rate.
Computing the MLD problem for general stabilizer codes has been shown to be a $\sharp P$-hard \cite{MP_sharp_P_hard}.
We will go into more detail in the following section \ref{ssec:general_ML_decoder}. 

The second is the so called Minimum Weight Perfect Matching (MWPM) approach.
It has a lower success probability but can be solved more efficiently in general. 
Here certain properties of how errors interact with stabilizers are used to find the error operator with the highest probability of occurring. 
We will go into more detail in the section \ref{ssec:general_MWMP_decoder}. 

Note that we have shown that the number of possible syndromes scales exponentially with the distance of the code.
As such it becomes unfeasible to compute the look up table ahead of time. 
But under certain conditions\footnote{We will discuss those conditions a bit when we talk about Pauli frame tracking \textcolor{red}{put ref here}.} we need to be able to correct the errors in real time.
Therefore efficient decoding techniques become more relevant to enable an real time decoding and correction of occurred errors during QEC cycles. 


\subsection{Maximum Likelihood Decoding}\label{ssec:general_ML_decoder}

In this section we will formalize the Maximum Likelihood Decoding (MLD) strategy for stabilizer codes \cite{mld_decoding_intro_luis}.
The MLD problem is finding the logical coset\footnote{We can and will once again disregard overall phase of this logical operator.} of the stabilizer group $\bar{L_MLD}\mathcal{S}$, in the centralizer with the corresponding syndrome, with the highest total probability of occurring, given our error model.
We can formalize this as
\begin{equation}
    \bar{L}_{\text{MLD};\sigma} = \arg \max_{\bar{L} \in  {\bar{\mathcal{P}}}} \text{Pr}(E_\sigma \bar{L} \mathcal{S}), 
\end{equation}
where $\bar{L}_{\text{MLD}}$ is the logical component of the error/recovery, 
$\text{Pr}(E_\sigma \bar{L}\mathcal{S})$ is the probability of the cosets of the stabilizer group $\mathcal{S}$, 
where the exact cosets is determined by the canonical error $E_\sigma$ corresponding to the syndrome and the logical component $\bar{L} \in {\bar{\mathcal{P}}}$.

We can expand this equation in terms of probabilities of individuals Pauli operators as 
\begin{eqnarray}
    \bar{L}_{\text{MLD};\sigma} = \arg \max_{\bar{L} \in  {\bar{\mathcal{P}}}} \sum_{S \in \mathcal{S}} \text{Pr}(E\bar{L}S). 
\end{eqnarray}
The number of operators in this summation grows exponentially with the distance of the codes as the number of stabilizer elements of the stabilizer group grows exponentially\footnote{
    See section \ref{sec:kitaev_surface_code} where we discuss in detail how the number of generators of the stabilizer group scale with distance of the code. 
} with the distance of the code.

Note that by taking all possible operators into account the ML decoder takes the degeneracy of errors into account.
As such the MLD finds the coset with the highest combined probability of all errors, not only the error with the highest probability.
Therefore even if we have a single error in a different coset that has the highest probability of occurring, MLD will find the coset with the overall highest probability.
This is a crucial difference to the following MWPM decoder, as described in section \ref{ssec:general_MWMP_decoder}.

The problem of determining the correct logical component using MLD has been shown to be $\#$P-hard for arbitrary stabilizer codes \cite{MP_sharp_P_hard}.

% make the following more precise, which models and why?
We can map the MLD problem of stabilizer codes to the partition function of classical disordered spin models \cite{topological_quantum_memory}. 
Here we map the summation over different Pauli operators to the partition function of a classical disordered spin model.
Dependent on the specific model and its symmetries, some of those classical disordered spin models can be solved exactly \cite{topological_quantum_memory}\cite{fundamental_thresholds}.
For those we can identify a phase change in the model at the threshold probability, which we will discuss in section \ref{ssec:threshold_theorem}. 
And some of them have been shown to computational efficiently solvable \cite{efficient_decoding_algorithm}. % is this classical disordered spin? because we use even more here

If we can solve the MLD problem exactly, we can calculate the success rate of the MLD $p_{\text{success}}$ exactly using 
\begin{equation}
    p_{\text{success}} = \sum_{\sigma \in \{0,1\}^m} \text{Pr}(\bar{L}_{\text{MLD};\sigma}\mathcal{S}).
\end{equation}

\subsection{Minimum Weight Perfect Matching}\label{ssec:general_MWMP_decoder}

The second decoding strategy is Minimum Weight Perfect Matching (MWPM)\cite{pymatching}\cite{pymatching_loger_paper}.
We can summarize the goal of MWPM as finding the error operator with the highest probability of occurring given the observed syndrome.
Note that as we determine a single operator we are not taking the degeneracy of possible errors into account or even the possibility that different logical classes/cosets could have multiple errors with the lowest weight\footnote{
    Note as well that there exist approaches where the degeneracy is also included in such maximum probability decoders \cite{degeneracy_enhanced_mp_decoding}.
}.

The MWPM decoder is based on mapping the error model together with the stabilizers onto a Detector Error Model (DEM) which we then map onto a weighted graph, called the matching graph.

To get from the error model to the DEM, we observe how each error mechanism changes the bits of the syndrome.
We represent each bit of the syndrome as a detector and if the bit is changes we talk about a triggered detector.
We can then connect individual error mechanism to those detectors, by associating each error mechanism with bit of the syndrome they flip.
This correspondence to identifying with which generator of the stabilizer group the error mechanism anticommutes.
As such we can connect the error mechanism to detectors.

To keep tack of logical changes we label each edge with the additional information on how the associated error mechanism commutes with the logical observable.
For this we first need to fix a certain realization of logical operator, which is done by selecting the set $\bar{\mathcal{P}}$. 
We then label the each error mechanism that anticommutes with the logical observable, $1$ for anticommutes, $0$ for they commute.
This will be used to determine the effect of the determined most probable error at the end, by summing over the labels of all edges, where an odd sum (even) corresponds to the combined error anticommutating (commutation).

If each error mechanism triggers at most\footnote{For the surface codes errors on the edge only trigger one detector. To obtain an graphlike error model we can introduce a boundary node into our graph \cite{pymatching_loger_paper}.} two detectors, we can refer to the DEM as a graphlike error model.
We will at this point assume\footnote{This is a valid assumption for the surface code \cite{topological_quantum_memory}.} that we have such an error model.
For this to be a valid assumption we might need to decompose the $Y$-type errors into $X$-type errors and $Z$-type errors ($Y = i XZ$), to ensure the graphlike property of our error model.
We then solve the $X$-type and $Y$-type MWPM decoding problems independently.
% Talk about this beeing valid in CSS codes?

We can then map this graph like error model onto a matching graph by representing each detector as a node and each error mechanism as an edge.
Each edge represents an error $E$ and is weighted $w(E)$ by the probability of the error $\text{Pr}(E)$
\begin{equation}
    w(E) = \log \left( \frac{1-\text{Pr}(E)}{\text{Pr}(E)}\right)
\end{equation}
and each edge connects the nodes, which represent the detectors that the associated error mechanism triggers. 

The MWPM decoder now tries to find the most probable error event consistent with the syndrome.
Finding a error consistent with the syndrome correspondence to finding a set of edges, such that all triggered detectors have an odd number of incident edges and non triggered detectors have an even number of incident matches.
And to choose the most probable of such configurations we need to find the set of edges, that fulfills the above condition with the lowest total weight $\sum_E w(E)$. 
Note at this point that we are following here the definition of an minimum weight embedded matching, which is a variation of the MWPM problem and is the basis of PyMatching the MWPM decoder which we are going to use \cite{pymatching_loger_paper}. 

After finding such a set the decoder then returns the prediction if the logical observable commutes or anticommutes with the predicted error, by combining the labels on the predicted edges. 
As such it predicts the change of the logical observable due to the error, including the canonical representation of the syndrome and the logical component.
We could also use the configuration of edges to reconstruct the predicted occurred error.
As discussed in section \textcolor{red}{ref to pauli frame tracking} this will be not necessary.
% I could spent some time here elaborating the true and complete difference and why what is useful but his might make sense to do at a later point.
% prediction of observable instead of prediction of logical component in the coset

The use of the detectors which just represent changes of bits in the syndrome instead of the absolute value of the syndrome, relates to the isomorphism between the codespace and the other orthogonal subspaces of the Hilbert space.
As we discussed in section \ref{ssec:syndrome} each subspace associated to a syndrome is isomorphic to the codespace. 
Given that we are only interested in the change of the syndrome, we can use this MWPM decoding setup, to return corrupted codeword states of isomorphic spaces to their original state.
We would still get the same prediction as if we would have started in a state belonging to the codespace, assuming the same error occurred.
We will discuss this principle in more detail in section \ref{}\textcolor{red}{Put reference to pauli frame tracking here}.

For now lets close this comparison between MWPM and ML decoding by once again pointing out that MWPM find the most probable single error event, while ML finds most likely coset and as such the most likely occurred logical operation.

\section{Kitaev Surface Code}\label{sec:kitaev_surface_code}

The specific QEC code from the class of the stabilizer codes we are using is the (unrotated) surface code \cite{kitaev_surface_code}, \cite{other_surface_code_paper}, which is based on the toric code \cite{kitaev_toric_code}.
The difference between the toric code and the surface code is the change from periodic boundary conditions to open boundaries for the surface code.
The above papers and the paper by Dennis \cite{topological_quantum_memory} are the references of this chapter.

The surface code is a well established QEC code, that is used due to it advantageous properties.
One of those advantageous properties is that the surface code only requires next neighbor interactions.


To visualize the surface code of distance $d$, we arrange $n=d^2 + (d-1)^2$ physical qubits on a $d \times d$ lattice,
by placing the qubits on the nodes connecting the sites of the lattice.
Note that for the surface code with open boundary conditions we additionally remove the notes connecting the upper most lattice sites to form a rough boundary.
The resulting constructing can be seen in figure \ref{fig:surface_code_explaination} a), with the associated indexing we are using for the qubits.

As described in section \ref{sec:stabilizer_formalism} to define a QEC code of the class of stabilizer codes, we just need to define the stabilizer group of the code.
We can define the stabilizer group of the surface code by defining a set of generators of the stabilizer group which split up into site and plaquette stabilizers\footnote{
    They are generators of the group but we only refer to them as site and plaquette stabilizers to keep the text short.
}.
The site stabilizers are placed on the sites of the lattice and act (nontrivial) on each physical qubit sitting on edges incident to this site with a Pauli $X$ operator.
The plaquette stabilizers are places in the middle of each cell and operate with a Pauli $Z$ operator on each physical qubit sitting on the edges of this cell.
The placement of both is visualized in figure \ref{fig:surface_code_explaination} b) and c).

We can formalize the describe of the site stabilizers as 
\begin{equation}
    S_{X,i} =  \bigotimes_{j \in A_i} X_j,
\end{equation}
where $A_i$ is the set of physical qubits that sit on the edges incident to the site.
It follows for the plaquette stabilizers, that we can express them as 
\begin{equation}
    S_{Z,i} =  \bigotimes_{j \in B_i} Z_j,
\end{equation}
where $B_i$ are the edges forming the cell of the plaquette stabilizer.

Note that the site and plaquette stabilizers in the bulk act on $4$ qubits each, whereas the generators on the edge only act on $3$ qubits each.

We can easily check that all generators commute because site and plaquette stabilizers always overlap on an even number of qubits, either $0$ or $2$. 
And site stabilizers trivially commute with all other site stabilizers. 
The same holds for plaquette stabilizers.

We can identify that we have found $m=2d(d-1)$ linear independent generators.
Therefore we can encode
\begin{equation}
    k = n-m = d^2 + (d-1)^2 - 2d(d-1) = 1
\end{equation} 
logical qubit.

The associated logical operators are the logical bit-flip $\bar{Z}$, a string of $Z$-type Paulis connecting the rough boundaries (top and bottom),
and the logical phase-flip $\bar{X}$, a string of $X$-type Paulis connecting the smooth boundaries (left and right site).
One representation of each is shown in figure \ref{fig:surface_code_explaination}.

\section{Fault Tolerance and Error Models}\label{sec:fault_tolerance}

In the previous section we implicitly assumed that errors only happen at a specific point in our quantum circuit.
Right after we encoded our state and then they stop occurring when we want to extract the syndrome and recover the state.
This is assumption is not well motivated as in realistic implementations, error can occur in every part of the circuit and with every gate applied.
Therefore we need to change our error model.
We will talk about the specific error models we will look at in this thesis in section \ref{ssec:error_models}, and the one that is described above is called the circuit level error model.

At this point we introduce the notion of fault tolerance in relation to QEC codes.
The concept of fault tolerance describes how our QEC code behaves under the condition that faults happen during its execution.
Given that all components of a circuit can fail, we can not expect of our QEC code to recover the original state.
For fault tolerance we change the requirements.
If we previously had a distance $d$ QEC circuit, that was able to correct $t$ errors, we now require of our fault tolerant circuit distance $d$ circuit, that $t$ faults lead to at most $t$ errors at the end of the QEC cycle.
As such the resulting state should be a correctable one with the same circuit given the noiseless assumption \cite{gottesmann_book}.
We refer as such a QEC as distance preserving and we will refer to the error remaining after the QEC cycle as the residual error.

One reason $t$ faults could lead to more than $t$ faults at the end of a circuit is so called error propagation, which we will talk about using the example of a CNOT gate.

\subsection{CNOT Gate}\label{ssec:cnot_gate}
As an example for error propagation and because we will use its properties later we will shortly discuss the CNOT gate.

It is a two qubit gate and implements the controlled NOT operation.
At especially important for the use case in this paper we look at how propagates errors $E$ \cite{gottesmann_book}.
As the CNOT operation is unitary and hermitian we can write
\begin{equation}
    \text{CNOT} \ E\ket{\Psi} = \text{CNOT } E\text{ CNOT} \ \text{CNOT} \ket{\Psi}.
\end{equation}
The propagated error is therefore $\text{CNOT } E \text{ CNOT}$.
We can identify that an error $E = X \otimes I$, will be propagated to become $X \otimes X$.
Here the first element of the tensor product is the error acting on the control qubit and the second is acting on the target qubit of the CNOT gate. 
As such the CNOT gate propagates $X$-type errors from the control bit onto the target bit. 
% Show a circuit diagram for CNOT gates!

We can identify such a propagation also in the other direction.
$E = I \otimes Z$, will become $Z \otimes Z$, which tells us that $Z$-type errors propagate from the target qubit onto the control qubit.

As such even if the CNOT gate itself is not faulty, if we an error is propagated through a CNOT gate, the number of errors in the circuit is increased.
And the CNOT gate propagates errors in both directions from control onto the target and vice versa.

\subsection{Threshold Theorem}\label{ssec:threshold_theorem}

The cornerstone of QEC codes and there usefulness is based on the threshold theorem, which states that even under the assumption that all components are noisy we can still arbitrary reduce the logical error rate of the logical qubit, given that the error rates of the noisy components is below a certain threshold \cite{gottesmann_book},\cite{topological_quantum_memory}.
As such we can if we the physical error rate is below the threshold we can reduce the logical error rate by using more resources.
This can be done for surface codes by increasing the distance $d$ of the code.
The overhead in the number of qubits has been shown to be polynomial \cite{topological_quantum_memory}.

Based on this theorem we not need to realize perfect qubits in an experimental setting to be able to realize arbitrarily long quantum computations
Instead we can use QEC codes to suppress the logical error rates to levels where long quantum computations become feasible.  
This makes quantum computational possible as awe do not need to create perfect qubits, just enough decent qubits, to enable quantum computing.

% A bit useless
Note that will below the threshold each additional qubit reduces the logical error probability, above the threshold the logical probability actually rises for each qubit added.

The threshold is QEC and error model dependent.
The threshold is an important quantity of a QEC codes. 
Codes with higher threshold can be used with less optimal qubits and therefore might enable quantum computation where other QEC codes still fail. 

The threshold can be determined through mapping to statistical physics models, as alluded to in section \ref{sec:ML_decoding}. 
In those models we can identify a phase change between an ordered and unordered state at the threshold \cite{fundamental_thresholds},\cite{topological_quantum_memory}.
 
We can also determine the threshold using numerical simulations of the QEC code with an associated error model.
This will be the approached used in this thesis.
To determine the threshold we compute the logical error rate for different distances and different physical error rates. 
Above the threshold the higher distance codes should have a higher logical error rate, whereas there increased distance results in a lower logical error rate below the threshold.
Given that we want to determine the location of a phase transition, when we want to locate the threshold, we can use finite-size scaling analysis to determine the value of the threshold give the data.

We can also state an asymptotic expectation for the scaling below the threshold.
We will assume at this point that we will only choose codes with odd distances $d$ for practical purposes.
In addition we assume that every noise process has a probability proportional to $p_{\text{phy.}}$ 
\begin{equation}
    \text{Pr}(E) \propto p_{\text{phy.}}.
\end{equation}
As our distance $d$ code is guaranteed to correct $t=(d-1)/2$ faults, assuming it is fault tolerant.
Therefore we need at least $t+1$ noise processes to create a logical error.
We Should expect the asymptotic behavior for physical noise rates towards zero to behave like
\begin{equation}
    \lim_{p_{\text{phy.}}\rightarrow 0} p_{\text{log}} \approx p_{\text{phy.}}^{t+1} = p_{\text{phy.}}^{(d+1)/2}.
\end{equation}
Here we discarded terms of higher order in $p_{\text{phy.}}$ as they are suppressed by the asymptotic limit.

\subsection{Error Models}\label{ssec:error_models}

The threshold of the QEC code depends also on the chosen error model.

\subsubsection{Code Capacity Noise Model}\label{sssec:code_capacity_error_model}

For the code capacity noise model we apply independent errors onto the data qubits after we have encoded the logical qubits.
As such we implicitly assume that all gates in the encoding, syndrome extraction and associated ancilla qubits and measurements are faultless.
This is the simplest noise model \cite{mld_decoding_intro_luis}. 

This model can be realized by applying a depolarizing, bit-flip or phase-flip noise channels.

\subsubsection{Circuit Level Noise Model}\label{sssec:circuit_level_noise}

The other noise model we will use is the so called circuit level noise model.
This is designed to be more realistic, as we model all gates and measurements to be faulty.
In general one assigns errors to every operation which include state preparation, single- and two-qubits gates, idle location and measurements.
This means that our whole decoding process is faulty as well, as we will see and discuss later.
Only fault tolerant QEC codes show distance preserving properties under this error model.

Note here that these errors propagate through the circuit, as discussed in section \ref{ssec:cnot_gate}, during the QEC cycle, which leads to correlated errors being spread over multiple qubits and having influence on multiple syndromes.
