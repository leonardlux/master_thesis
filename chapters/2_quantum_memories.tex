Why do we bother to develop quantum computing?

Quantum computing is interesting because it offers the possibility to use quantum phenomena like superposition and entanglement in algorithms.
Such algorithms can solve some problems efficiently for which we do not yet know an efficient classical algorithm.
A famous examples is the problem of integer factorization, as proposed by Shor \cite{shor_factoring}. % because the problem of integer factorziation was the basis for a lot of encryption
Other examples are the grover search algorithm \cite{grover_search_algo} or this \cite{chemistry_example_algo} proposed algorithm to calculate the energy of atoms and molecules.

To realize quantum computations we first need a quantum state in which we can store our quantum information, analogous to the (classical) bit of a (classical) computer.
Such a state is called a quantum bit (qubit)\footnote{

To implement the proper algorithms we also need to be able to do operations on the quantum information, but the implementation of those is out of scope of this thesis. 
We will focus on maintaining the quantum information contained in one qubit over time, a so called quantum memory experiment.

To mitigate the influence of errors and safeguard the quantum information against errors, quantum error-correction (QEC) codes were developed. %should put same papers here
Quantum error-correction codes are different from classical error-correcting codes, because we have to take into account that by measuring we can destroy superpositions (Wavefunction collapse).
Moreover where there is only one type of error in classical computers, the bit flip, quantum computers can have a multitude of different errors, as will be discussed in section \ref{sec:pauli_errors}.

In QEC codes we take active measures to correct the quantum memory and remove errors from the qubits. 
This is done by encoding one\footnote{We can also encode more than one qubit at a time, but this is also not the focus of this work.} (logical) qubit in many (physical) qubits. 
Using the redundancy in those qubits we can extract some information, which does not tell us anything about the logical state, but can be used to infer which errors occurred.
And based on this information we can take actions to correct those occured errors \cite{terhal_qec_formalism}.

We will got into more detail in section \ref{sec:stabilizer_formalism}, where we focus on stabilizer codes, which are a class of quantum error-correcting codes and which we are gonna use in this thesis.

\section{Stabilizer Codes and their Structure}\label{sec:stabilizer_formalism}

One class of QEC codes are the stabilizer codes \cite{gottesmann_book},\cite{gottesmann_thesis}.
This section is mainly based by Gottesmann works, as cited above, and the work of Terhal \cite{terhal_qec_formalism}.

In this and in all the following chapters we limit ourselves to Pauli operators as possible errors. 
The argumentation why this is assumption valid is following in section \ref{sec:pauli_errors}.

The goal of this chapter is to introduce the structure and formalism behind stabilizer codes.
We want to place a special focus on the group structure behind stabilizer codes to motivate the later chapter about possible decoder approaches. 

\subsection{Stabilizer Group}

The stabilizer formalism is based around a subgroup of the Pauli group $\mathcal{P}_n$ called the stabilizer group $\mathcal{S}$. 
The reason for the name becomes obvious in the next subsection \ref{ssec:codespace}, when we talk about the codespace.

By choosing a certain set of Pauli operator one defines the stabilizer code.
We will later in section \ref{sec:kitaev_surface_code} discuss a specific choice but we first take a general view upon stabilizer codes. 
One can freely choose which operators to use to define a stabilizer group with the following limitations for the resulting stabilizer group.
It must be an Abelian subgroup of the $n$-qubit Pauli group $\mathcal{P}_n$ and it is not allowed to contain $-I$. 

Many of the following definitions will be formulated by only taking a set of $m$ linear independent generators of the stabilizer group, which we will label as $S_i$, such that the stabilizer group can be written as $\langle S_1, S_2, ..., S_m\rangle= \mathcal{S}$.
To shorten the notation even further we will refer to the set of linear independent generators of the stabilizer group by $\mathcal{S}_G$.
We will also refer to the linear independent generators of the stabilizer group by generators to shorten the following text.


\subsection{Codespace}\label{ssec:codespace}% rework this headline 

As described earlier the basic idea behind QEC codes is to encode the logical information of $k$ logical qubits into $n$ physical qubits.  
The space in which the logical states exist is called the codespace $\mathcal{L}$, which is a subspace of the full $n$-qubit Hilbert space $\mathcal{H}_{2^n}$. 

Using the stabilizer group defined in the previous subsection, we can define the codespace $\mathcal{L}$ as being the space spanned by all states $\ket{\Psi}$ which are invariant under all elements of the stabilizer group.

\begin{equation}
    \mathcal{L} = \left\{  \ket{\Psi} \in  \mathcal{H}_{2^n} : S \ket{\Psi} = \ket{\Psi} \forall S \in \mathcal{S} \right\}
    \label{eq:defintion_codespace_whole_group}
\end{equation}

Given that each element of the stabilizer can be generated by our set of generators $\mathcal{S}_G$, we can rewrite this definition as 

\begin{equation}
    \mathcal{L} = \left\{  \ket{\Psi} \in  \mathcal{H}_{2^n} : S \ket{\Psi} = \ket{\Psi} \forall S_i \in \mathcal{S}_G \right\}.
    \label{eq:defintion_codespace_generators}
\end{equation}

Given that we have $n$ physical qubits into which we encode the logical information and used a stabilizer group with $m$ linear independent generators, we can encode $k=n-m$ logical qubits into the resulting codespace. 
The resulting codespace is therefore a $2^k$ dimensional Hilbert space.

We will refer to the states which are in the codespace, as codewords $\ket{\Phi}\in \mathcal{L}$.

Here we note explicitly that acting with an element of the stabilizer group on any codeword is equivalent to act with the identity
\begin{equation}
    S \ket{\Phi} = \ket{\Phi}.
    \label{eq:stabilizer_equivalence}
\end{equation}
We will refer to this intrinsic property of stabilizer codes by calling it the stabilizer equivalence.

\subsection{Syndrome}

Given that we now have a defined the codespace, we can detect if a given eigenstate of the stabilizer group\footnote{
    We will elaborate in section \ref{sec:pauli_errors} why we only care about eigenstates of the stabilizer group.
} is in the codespace by determining the eigenvalues of the generators of the stabilizer groups.
Based on the eigenvalues we can construct an $m$-bit string, the so called syndrome $\sigma(\ket{\Psi}) \in \mathbb{Z}_2$, using
\begin{equation}
    S_i \ket{\Psi} = (-1)^{\sigma_i} \ket{\Psi}.
    \label{eq:state_syndrome}
\end{equation}
If the state is a $+1$ ($-1$) eigenstate of $i$-th generator the $i$-th bit of th syndrome $0$ ($1$).
Therefore the syndrome of the state tell us if the state belong to the codespace, all bit equal $0$, or if it lies outside of it.
We will refer to the all $0$ syndrome as the trivial syndrome.

Note that given we have $m$ generators, there are $2^m$ possible different syndromes.


We can also construct a similar syndrome string for elements of the Pauli group $E\in \mathcal{P}_n$, here take the element $E$ to be represent an error. 
This so called error syndrome $\sigma(E) \in \mathbb{Z}_2$ is constructed by first applying the operator $E$ to any codeword $\ket{\Phi}$.
We can then use the definition of the syndrome of a state to assign a syndrome to the error
\begin{equation}
    \sigma(E) = \sigma(E\ket{\Phi}); E \in \mathcal{P}_n; \Phi \in \mathcal{L}.
    \label{eq:error_syndrome}
\end{equation}
The resulting syndrome thus tells us how the operator $E$ would change the syndrome of a codeword. 
Note that the choice of the specific codeword does not matter, as long as it is part of the codespace.

Looking at the definition of the $i$-th bit of the error syndrome, 
\begin{equation}
    S_i E \ket{\Phi} = (-1)^{\sigma_i} E S_i \ket{\Phi}.
\end{equation}
we can identify that the error $i$-th bit of the error syndrome represents the commutation relation between the error and the $i$-th generator. 
If they commute (anticommute) the error syndrome of the $i$-th bit has the value $0$ ($1$).
Given that both the error operator and the generator are Pauli operators, they either commute or anticommute.

\subsection{Centralizer, Cosets and Group Structure}\label{ssec:intro_centralizer} % find a better title 

As eluded to in the introduction, there is a certain group structure to both the Hilbert space and Pauli group when using the stabilizer formalism which we will now investigate.

Given the observed importance of commutation relations between operators and stabilizers regarding the syndrome in the previous chapter, we are motivated to look at the centralizer\footnote{The centralizer is the same as the normalizer for group made out of Paulis, because Paulis either commute or anticommute. Some people refer to this group as the normalizer of the stabilizer group.} of the stabilizer group $\mathcal{C}(\mathcal{S})$
\begin{equation}
    \mathcal{C}(\mathcal{S}) = \left \{  P \in \mathcal{P}_n | P S_i = S_i P\  \forall S_i \in \mathcal{S}_G \right \}.
\end{equation}
The centralizer of the stabilizer group contains all the elements of the Pauli group which commute with all elements of the stabilizer group.
To shorten the following text we will refer to the centralizer of the stabilizer group as centralizer and in case we talk about the centralizer of a different group, we will specify it.

We can deduce that all elements in the centralizer have the trivial syndrome, because they all commute with all generators.
Note that having a trivial syndrome is equivalent with the operators beeing not detectable. % TODO expand this thought


Given that the stabilizer group itself is Abelian, the centralizer contains the stabilizer group as a subgroup $\mathcal{S} \subset \mathcal{C}(S)$.
The centralizer also contain the sets $-\mathcal{S}, \pm i \mathcal{S}$.

But if we think about the operators in the context of possible errors on the code, or operators applied to the code, and not measurements to determine the syndrome, we can identify the phase as a global phase and are therefore not long interested in it.
Following the formalism in \cite{gottesmann_book} we will indicate the set in which we ignore the overall phases of the operator by a hat symbol
\begin{equation}
    \hat{\mathcal{P}}_n \cong \mathcal{P}_n / \{I, iI, -I, -iI\}.
\end{equation}


Coming back to the topic at hand, we can observe that the centralizer $\hat{\mathcal{C}}(\mathcal{S})$ contains the stabilizer group $\hat{\mathcal{S}}$ 
and all other elements of the Pauli group $\hat{\mathcal{P}}_n$ which have the trivial syndrome.

Looking at the centralizer without the stabilizer $\hat{\mathcal{C}}(\mathcal{S})\setminus \hat{\mathcal{S}}$ we find an interesting set of operators.
They have the trivial error syndrome, therefore they do not change the syndrome of the state if they act on the state. 
But by definition they do not map the same codeword onto itself, otherwise they would belong to stabilizer group, which we have excluded here.
Therefore we can conclude that they map codeword states to other codeword states.
From this argumentation we can conclude that we have found a set of operator which realize the logical\footnote{
    Here we refer to the logical operators which do not act trivially ($\bar{X}, \bar{Z}, \bar{Y}$). 
    The logical identity $\bar{I}$ is not changing the logical state and therefore is not changing between codewords and can be realized by any element of the stabilizer group.
} operators on the codespace.

Those operators could be applied on purpose to change from one codeword to another, which means to apply a logical operator on the encoded qubit.

But if they happen through noise acting on the system, the logical information contained in the encoded qubit would change.
This would represent an undetectable, because it has no syndrome and therefore not be distinguished from a valid encoded state, error. 

Therefore the set of all possible realizations of the non identity logical operators is identical to the set of undetectable errors.

\subsubsection{Distance}
% Distance
At this point we can introduce an characteristic quantity of QEC codes. % more general of QEC codes
The distance $d$ which represent the minium weight\footnote{The weight of an operator is the number of qubits on which the operator acts non trivially.} that of a logical operator/undetectable error.
\begin{equation}
    d = \min_{E\in \mathcal{C}(\mathcal{S})\setminus \mathcal{S}} |E|
\end{equation}
A distance $d$ code can correct $t$ errors, with 
\begin{equation}
    t =  \left \lfloor \frac{d-1}{2} \right \rfloor.
    \label{eq:correctable_errors}
\end{equation}
The distance give us therefore an lower bound of how good our QEC code can perform. %TODO rewrite this sentence I do not like it yet!

And now we have all necessary quantities to introduce a notation characterizing stabilizer codes.
We refer to stabilizer codes by their set of characteristic number as $[[n,k,d]]$ codes.
$n$ being the number of physical qubits, $k$ being the number of encoded qubits and $d$ is the distance of the code.

In subsection \ref{ssec:intro_centralizer} we introduced the centralizer. 
There we discovered that the centralizer contains the realizations of the logical operators on the encoded qubits (and the stabilizer group, 
which can be seen as the representation of the identity operator).
In this subsection we will show the underlying group theoretic structure of the centralizer, following the work of Gottesmann in \cite{gottesmann_book} (Proposition 3.10).
Understanding this structure will become especially helpful, when we discuss the decoding problem. %TODO ref the correct chapter here.

Central to this argumentation is that if and only if two operators from the centralizer $L_1, L_2 \in \mathcal{C}(\mathcal{S})$ act the same way on each codeword $\ket{\Phi}$, 
implying that they realize the same logical operation, 
they are in the same cosets of the stabilizer group.

To proof that we start with our assumption about the two operators
\begin{equation}
    L_1 \ket{\Phi}=L_2 \ket{\Phi}; L_1, L_2 \in \mathcal{C}(\mathcal{S}); \forall \ket{\Phi} \in \mathcal{L}
\end{equation}
which we can rearrange to 
\begin{equation}
    M \ket{\Phi} = +1\ket{\Phi}  ; \forall \ket{\Phi}\in \mathcal{L}
\end{equation}
by defining $M =  L_1^\dag L_2$.

We can see that the operator $M$ is by definition (see eq. \ref{eq:defintion_codespace_whole_group}) an element of the stabilizer group, because all codewords are in its $+1$ eigenstates.

If we rearrange the definition of $M$ we get the that $L_1$ and $L_2$ are in the same coset of the stabilizer group
\begin{equation}
    L_2 = L_1 M \text{ with } M \in \mathcal{S}. 
\end{equation}

Thus we can conclude that all the different realizations of the same logical operator are contained in the same coset of the stabilizer group.


All elements of the same coset of the stabilizer group act therefore the same on all codewords.
They all implement the same logical operation. 

% quotient group
Looking at the quotient group $\mathcal{C}(\mathcal{S})/\mathcal{S}$ we find that it contains all possible distinct logical operations and is isomorphic to the Pauli group $\mathcal{P}_k$.
This we can write as 
\begin{equation}
    \mathcal{C}(\mathcal{S})/\mathcal{S} \cong \mathcal{P}_k.
\end{equation}
We can therefore realize logical operations on the encoded state by using an operator of the corresponding\footnote{
    We have to fix this correspondence in the beginning, see the note section following.
    } 
coset of the stabilizer group in the centralizer.
The exact choice of element from the coset is not important, all elements are logical equivalent and they only differ by an element of the stabilizer group, 
which by stabilizer equivalence (eq. \ref{eq:stabilizer_equivalence}) is to say, they do not differ. 




% Note about freedom in assigning logical operators 
Note as well that we have some freedom of assigning the logical operators to the different cosets.
We just have to make sure that the commutations relations between the realizations of the logical operators are correct.
(As an example it is trivial to switch the logical $\bar{X}$ with the logical $\bar{Z}$ operator.)
