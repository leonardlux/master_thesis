\section{Pauli Operators and General Errors}\label{sec:pauli_errors}

Before we discuss how to correct error, we first need to discuss what types of errors can occur and how we can model them, following \cite{terhal_qec_formalism}, \cite{gottesmann_thesis} and \cite{intro_to_qec}.

Towards this goal we first introduce the Pauli group.
Afterwards we will discuss how general quantum noise/errors behaves and how we can connect them to the Pauli group.

\subsection{Pauli Group}\label{ssec:intro_pauli_group}

The Pauli group $\mathcal{P}_n$ is made up of tensor products of the one qubit Pauli group without phase $P \in \hat{\mathcal{P}}_1 = \{ I,X,Z,Y \}$ 
\begin{equation}
I =
\begin{pmatrix}
1 & 0 \\
0 & 1
\end{pmatrix}, \qquad
X =
\begin{pmatrix}
0 & 1 \\
1 & 0
\end{pmatrix}, \qquad
Y =
i
\begin{pmatrix}
0 & -1 \\
1 & 0
\end{pmatrix}, \qquad
Z =
\begin{pmatrix}
1 & 0 \\
0 & -1
\end{pmatrix}
\end{equation} 
together with an overall phase $\{\pm1,\pm i\}$
\begin{equation}
    \mathcal{P}_n = \{c P_1\otimes P_2 \otimes ... \otimes P_n \mid c \in \{\pm 1, \pm i\}; P_i \in \hat{\mathcal{P}}_1 \forall i \}.
\end{equation}

% Make the 
Note that a global phase of a state has no physically observable consequence and as such we can omit it in most cases.
Following this argumentation it can be sometimes useful to omit the overall phase of an operator in some cases.
When we want to talk about the Pauli group without a phase we will refer to it as $\hat{\mathcal{P}}_n$.

At this point we introduce the weight, denoted $|P|$, of a Pauli operator $P\in\mathcal{P}_n$ as the number of non-identity Paulis in the tensor product that defines $P\in\mathcal{P}_n$.
This quantity will become interesting when we talk about the distance of codes and what errors are guaranteed to be correctable in section \ref{sssec:distance_stabilizer_formalism}. 

The Pauli operators are closed under multiplication, especially $Y = i X Z$.
Each Pauli squared, disregarding the phase, is identical to identity
\begin{equation}
    I = X^2 = Z^2 = Y^2.
\end{equation} 
Resulting from this any tensor product of Paulis, is also its own inverse, when we disregard the overall phase
\begin{equation}
    \tilde{P} \tilde{P} = I \ \forall \tilde{P} \in \hat{\mathcal{P}}_n.
    \label{eq:pauli_squared_is_identity}
\end{equation}
Taking the phase into account the square of an element of the Pauli group is $\pm I $, where the sign depends on the overall phase.  
This property becomes important when we want to find a recovery for a given Pauli error in section \ref{ssec:stabilizer_summary_qec_cylce}. 
Where we will use that we can recover from the error $P$ by reapplying the same Pauli. 
In the process we might change the overall phase of the state but otherwise act with the identity, under the assumption that we where able to correctly identify error and could therefore reapply it.

Another property of the Pauli group is that all their elements either commute or anticommute.
For the one qubit Pauli group all non trivial pairings anticommute, we will use $\{A,B\}$ to indicate the anticommutator, 
\begin{equation}
    \{X,Z\} = \{X,Y\} = \{Z,Y\} = 0.
\end{equation} 
Elements of the $n$ qubit Pauli group $\mathcal{P}_n$ can also non trivially commute if they have an even number of anticommute elements. 

Each Pauli operator splits the Hilbert space in half, with each half either being associated with the eigenvalue $+1$ or $-1$.
A fact that we use to define the codespace.

We can use the $n$-qubit Pauli group as a basis to express any quantum channel. 
As it spans the space of all $n$-qubit linear operations.
% Should I here discuss more the superoperators: maarten commentet this here in feedback 1
Which will become important when we discuss general errors.

We will refer to $X$-type Pauli operator as bit-flip, as this error flips the $\ket{0}$ state to the $\ket{1}$ state and vice versa 
\begin{equation}
    X \ket{\Psi} = X(\alpha \ket{0} + \beta \ket{1}) = \beta \ket{0} + \alpha \ket{1}.
\end{equation}
And we will refer to the $Z$ Pauli operator as phase-flip, as it maps the $\ket{1}$ basis state to $-\ket{1}$, while leaving the $\ket{0}$ basis state unchanged
\begin{equation}
    Z \ket{\Psi} = Z(\alpha \ket{0} + \beta \ket{1}) = \alpha \ket{0} - \beta \ket{1}.
\end{equation}

The $Y$ Pauli is just bit- and phase-flip applied with an additional overall phase of $i$.

\subsection{Stabilizer Group}

The stabilizer formalism is based on some choice of subgroup of the Pauli group $\mathcal{P}_n$ called the stabilizer group $\mathcal{S}$. 
The reason for naming the subgroup stabilizers becomes clear in the next subsection \ref{ssec:codespace}, when we talk about the codespace.

By choosing the stabilizer group one defines the specific stabilizer code and the specific encoding of the QEC code.
We will later in section \ref{sec:kitaev_surface_code} discuss a specific choice. 
In the following we take a general view upon stabilizer codes. 
One can freely choose which operators to use to define a stabilizer group, with the following limitations for the resulting stabilizer group.
It must be an Abelian subgroup of the $n$-qubit Pauli group $\mathcal{P}_n$ and it is not allowed to contain $-I$. 

Many of the following definitions will be formulated by taking a set of $m$ linear independent generators of the stabilizer group, which we will label as $S_i$, such that the stabilizer group can be written\footnote{
    Here the angled brackets $\langle S_1, ..., S_m \rangle= \mathcal{P}_N$ denotes that the group is generated by the set inside of the brackets.
    The group $\mathcal{P}_n$, then consists off all finite products of elements of this denoted set and of their inverses. 
    } as $\langle S_1, S_2, ..., S_m\rangle= \mathcal{S}$.
We will label to the set of linear independent generators of the stabilizer group by $\mathcal{S}_G$ and refer to them simply as generators.
The reason why we choose to only use the set of operators will become clear in section \ref{ssec:syndrome}. 


\subsection{Codespace}\label{ssec:codespace}% rework this headline 
% Todo, maarten placed a k here... understand why?

As described earlier the basic idea behind QEC codes is to encode the logical information of $k$ logical qubits into $n$ physical qubits.  
The space in which the logical states lie, called the codespace $\mathcal{L}$, is a subspace of the full $n$-qubit Hilbert space $\mathcal{H}_{2^n}$. 

Using the stabilizer group defined in the previous subsection, we can define the codespace $\mathcal{L}$ as the space spanned by all states $\ket{\Psi}$ which are invariant under all elements of the stabilizer group,
or given that each element of the stabilizer can be generated by the set of generators $\mathcal{S}_G$, we can also use this set 

\begin{align}
    \mathcal{L} &= \left\{  \ket{\Psi} \in  \mathcal{H}_{2^n} : S \ket{\Psi} = \ket{\Psi} \forall S \in \mathcal{S} \right\}
    \label{eq:defintion_codespace_whole_group} \\
     &= \left\{  \ket{\Psi} \in  \mathcal{H}_{2^n} : S \ket{\Psi} = \ket{\Psi} \forall S_i \in \mathcal{S}_G \right\}.
    \label{eq:defintion_codespace_generators}
\end{align}

Given that we have $n$ physical qubits into which we encode the logical information and given we use a stabilizer group with $m$ linear independent generators, we can encode $k=n-m$ logical qubits into the resulting codespace. 
Which we can understand by following the argumentation:
Each of the $m$ linear independent generators splits splits the $2^n$ dimensional Hilbert space into two subspaces associated with the eigenvalues $+1$ and $-1$.
The resulting codespace is therefore a $2^k=2^n/2^m$ dimensional Hilbert space, which is characterized by being the $+1$ eigenspace of all generators (and all stabilizers).

We will refer to the states in the codespace, as codewords or logical states $\ket{\Phi}\in \mathcal{L}$.

Here we note explicitly that acting with an element of the stabilizer group on any codeword is equivalent to acting with the identity on them,
given that the codewords states are defined as being $+1$ eigenstates of all elements of the stabilizer group
\begin{equation}
    S \ket{\Phi} = \ket{\Phi}.
    \label{eq:stabilizer_equivalence}
\end{equation}
Therefore the codeword states are invariant under the action of the stabilizer group.

We can extend this notion to operators.
Two operators are called stabilizer equivalent if they differ by a stabilizer element. 
Since every codeword is invariant under action of the stabilizer group, such operators have the same action on all codeword states.


\subsection{Syndromes and Error Subspaces}\label{ssec:syndrome}

Now that we have defined the codespace, we can detect if a given eigenstate of the stabilizer group\footnote{
    Only such states are of interest, because they are the basis of our logical information. % Improve why only those states are relevant 
} is in the codespace or if it has left the codespace by occurrence of some error. 
We determine the eigenvalues of the generators of the stabilizer groups for the given state $\ket{\Psi}$.
Based on the eigenvalues we can assign the state a $m$-bit string, called the syndrome, $\sigma(\ket{\Psi}) \in \mathbb{Z}_2$, following 
\begin{equation}
    S_i \ket{\Psi} = (-1)^{\sigma_i} \ket{\Psi}\ \sigma_i \in \{0,1\},
    \label{eq:state_syndrome}
\end{equation}
If the state is a $+1$ ($-1$) eigenstate of $i$-th generator, the $i$-th bit of the syndrome takes the value $0$ ($1$).
By definition the state lies in the codespace if and only if it is the $+1$ eigenstate of all generators.
This corresponds to a syndrome consisting of an all-zero bit string.
We will refer to this syndrome as the trivial syndrome.
All states with non-trivial syndromes lie outside of our defined codespace. 

Note that given we have $m$ generators, there are $2^m$ possible different syndromes.

We can construct a similar syndrome string for elements of the Pauli group $E\in \mathcal{P}_n$. 
Note that we assumed that we can represent errors as elements $E\in \mathcal{P}_N$. 
To define the error syndrome $\tilde{\sigma}(E) \in \mathbb{Z}_2^m$ we imagine an error with operator $E$ acting on some codeword $\ket{\Phi}$.
Then we determine the syndrome of the resulting state $E\ket{\Phi}$ following the procedure above.
We define the resulting syndrome as the error syndrome of the operator $E$
\begin{equation}
    \tilde{\sigma}(E) \coloneqq \sigma(E\ket{\Phi}); E \in \mathcal{P}_n; \Phi \in \mathcal{L}.
    \label{eq:error_syndrome}
\end{equation}
Note that the error syndrome  $\tilde{\sigma}$ and the (state) syndrome $\sigma$ are formally different mathematical objects: 
the former takes Pauli operators as input, while the latter takes quantum states as input.
Given that the codeword has the trivial syndrome, we will neglect this mathematical difference\footnote{
    And we will refer to both syndromes with $\sigma(E)$ or $\sigma(\ket{\Psi})$.
    The mathematical object should be clear from context.
    } and we associate the constructed syndrome with the error operator. 

Looking at the definition of the $i$-th bit of the error syndrome, 
\begin{equation}
    S_i E \ket{\Phi} = (-1)^{\sigma_i} E S_i \ket{\Phi}.
\end{equation}
we can identify that the $i$-th bit of the error syndrome is defined by the type of commutation relation between the error and the $i$-th generator.
If the generator and the error operator commute (anticommute) the error syndrome of the $i$-th bit has the value $0$ ($1$).
Given that we only consider errors and generator as Pauli operators, commutation or anticommutation are the only two possibilities.

Note that the choice of the specific codeword does not matter to determine the syndrome of an error operator.
The error operators changes the syndrome of each codeword by its associated error syndrome. 
A key result is that it maps the whole codespace onto a different subspace where the eigenvalue for all stabilizers is no longer $+1$ but the syndrome of the error operator for each codeword.

This implies that the quantum information is preserved and just moved to a different subspace, from which it can be recovered.
And all of those $2^m$ subspaces are isomorphic to the codespace.
This means we can use any of those subspaces as codespace.
A fact that we are going to use in subsection \ref{ssec:cosets_of_the_centralizer}.

To change the codespace to the space associated with the syndrome $\sigma'$, we redefine the $i$-th generator $S_i$ to be $-S_i$ for each bit in the syndrome ${\sigma'}$ which takes the value $1={\sigma'}$. 
When we use the resulting set of linear independent generators to define the stabilizer group, the subspace corresponding to the syndrome ${\sigma'}$ using the previous set on linear independent generators, 
has now trivial syndrome and is therefore our chosen codespace. % this sentence is to long and cumbersome!
We will use this fact in section \ref{sec:pauli_frame_tracking} where we talk about Pauli Frame Tracking.

Summarizing we have $2^m$ subspaces which are isomorphic to the codespace, one for each syndrome. 
All of those subspaces are orthogonal to each other, because they have different eigenvalues/different syndromes.
Each of these subspaces is $2^k$  dimensional and encodes the $k$-logical qubits, where $k=n-m$.
Therefore we can decompose the full Hilbert space $\mathcal{H}_{2^n}$ into a direct sum of the $2^m=2^{n-k}$ subspaces, each one associated with a different syndrome.
The distinction of one quantum code space out of the $2^m$ possible isomorphic spaces is a classical choice that cen be altered during QEC, under certain conditions discussed in section \ref{sec:pauli_frame_tracking}.

\subsubsection{Centralizer}\label{sssec:centralizer_intro_and_defintion}

Motivated by the above observed importance of commutation relations between operators and stabilizers regarding the syndrome, we look at the so-called centralizer\footnote{Also known as commutant.} $\mathcal{C}(\mathcal{S})$ of the stabilizer group $\mathcal{S}$
\begin{equation}
    \mathcal{C}(\mathcal{S}) = \left \{  P \in \mathcal{P}_n \mid \  P S_i = S_i P\  \forall S_i \in \mathcal{S}_G \right \}.
\end{equation}
This centralizer contains all the elements of the Pauli group $\mathcal{P}_n$ which commute with all elements of the stabilizer group.
We will refer to the centralizer of the stabilizer group as just the centralizer as long as it is clear which stabilizer group $\mathcal{S}$ is considered.

We see that all elements in the centralizer have the trivial error syndrome, because they commute with all generators.
% Note that having a trivial syndrome is equivalent with the operators being not detectable. % TODO expand this thought

Given that the stabilizer group itself is Abelian, the centralizer contains the stabilizer group as a subgroup $\mathcal{S} \subset \mathcal{C}(S)$.
The centralizer also contains the sets $-\mathcal{S}, \pm i \mathcal{S}$. 
If we consider the operators in the context of possible errors acting on the space, rather than measurement operators used to determine the syndrome, we can treat the phase as a global phase and are therefore no longer concerned with it.
Following \cite{gottesmann_book} we will indicate the quotient group for which we ignore these four possible phases of the operators by a hat symbol
\begin{equation}
    \hat{\mathcal{P}}_n \cong \mathcal{P}_n / \{I, iI, -I, -iI\}.
\end{equation}
We will use the hat notation to indicate that the global phases are neglected for the corresponding group.

\subsubsection{Logical operators on the Codespace}\label{sssec:logical_operators_on_the_codespace_intro}

In this section we will identify the logical operators, operators which change encoded quantum information, as a subset of the centralizer. 
We define the logical operators as those operators which map the codespace back onto the codespace, while mapping codeword states to different\footnote{
    Here we refer to the logical operators which do not act trivially ($\bar{X}, \bar{Z}, \bar{Y}$).

} codeword states. 

We can observe that the phase normalized centralizer $\hat{\mathcal{C}}(\mathcal{S})$ contains the stabilizer group $\hat{\mathcal{S}}$ 
and all other elements of the Pauli group $\hat{\mathcal{P}}_n$ which have the trivial syndrome.

The set-difference between the centralizer and the stabilizer, denoted $\hat{\mathcal{C}}(\mathcal{S})\setminus \hat{\mathcal{S}}$, is the set of operators which map states from the codespace to different state from the codespace.
The underlying reasoning is those operators have the trivial syndrome, and therefore map the codespace back to the codespace, 
but by definition they do not map the same codeword onto itself, otherwise they would belong to stabilizer group, which we have excluded here.

\paragraph{Non-detectable Errors}

Logical operators change from one codeword to another, which means to apply a logical onto the encoded qubit.

But if they happen through noise acting on the system, the logical information contained in the encoded qubit, represented by a codeword, will change.
This represents an undetectable error. 
It is undetectable because our quantum error correcting code can only tell if a state is a valid codeword. 
It cannot distinguish whether a state is reached by original encoding/purposefully applied logical operators or by occurrence of errors, hence the name. 

Therefore the set of all possible implementations of non-identity logical operators is identical to the set of undetectable errors.

Note that all operators outside of the centralizer anticommute with at least one generator of the stabilizer group and therefore all of them are detectable. 
We will talk about this set of errors in subsection \ref{ssec:cosets_of_the_centralizer}.

Summarizing, we can detect errors by measuring the syndrome of the state acted upon with an error. 
We can only detect errors outside of the centralizer.
Errors operators which are part of the centralizer are either trivial correctable, in the case of stabilizer operators due to codeword being invariant under their action,
or they are undetectable errors for the case of logical operators.

\subsubsection{Distance}\label{sssec:distance_stabilizer_formalism}
We are ready not to introduce a characteristic number of a QEC codes. % more general of QEC codes
The distance $d$ of a QEC code is the minium weight of a non-identity logical operator/undetectable error.
\begin{equation}
    d = \min_{E\in \mathcal{C}(\mathcal{S})\setminus \mathcal{S}} |E|
\end{equation}
As introduced in section \ref{ssec:intro_pauli_group}, the weight $|E|$ of an operator $E\in \mathcal{P}_n$ is the number of qubits on which the operator acts non trivially.

The distance $d$ of a code is determined by the choice of the stabilizer group, since it directly determines the centralizer. 
The particular representatives from the set $\mathcal{C}(\mathcal{S})\setminus \mathcal{S}$ chosen to implement the logical operators do not alter the stabilizer group or the associated centralizer and therefore do not affect the distance.

Changing the phases of the generators of the stabilizer group does not modify the centralizer, since the phases do not affect commutation relations. 
Therefore the change of codespace and stabilizer group employed in Pauli frame tracking, see section \ref{sec:pauli_frame_tracking}, leaves the code distance $d$ unchanged.
It follows that the phase-normalized stabilizer group  $\hat{\mathcal{S}}$ uniquely determines the code distance.

A distance $d$ code can correct $t$ errors, with 
\begin{equation}
    t =  \left \lfloor \frac{d-1}{2} \right \rfloor.
    \label{eq:correctable_errors}
\end{equation}
This can be seen following the reasoning \cite{terhal_qec_formalism}: 

If we have a weight $t$ error $E$, our decoder can find a correction $R$ with the same syndrome, with at most weight $t$.
The combined operators $RE$ has then at most weight $2t$, which by the definition above, must be smaller than the distance $d$.
This combined operators has no error syndrome, therefore belongs to the centralizer and can by definition of the distance not be a non identity logical operator.
Therefore it is a stabilizer which cannot change a codeword and therefore the we can recover from this error.

\subsubsection{Logical Operations are Cosets of the Stabilizer Group in the Centralizer}

In subsection \ref{ssec:intro_centralizer} we introduced the centralizer. 
There we saw that the centralizer contains the implementation of the logical operations on the encoded qubits (and the stabilizer group, 
which can be seen as the representation of the identity operator).
In this subsection we review the underlying group structure of the centralizer, following \cite{gottesmann_book}.
Understanding this structure will become important for the decoding problem later on, see section \ref{sec:ML_decoding}.

The key observation is that if and only if two operators from the centralizer $L_1, L_2 \in \mathcal{C}(\mathcal{S})$ act the same way on each codeword $\ket{\Phi}$, 
\begin{equation}
    L_1 \ket{\Phi}=L_2 \ket{\Phi}; L_1, L_2 \in \mathcal{C}(\mathcal{S}); \forall \ket{\Phi} \in \mathcal{L}.

\end{equation}
they realize the same logical operation, 
they then are in the same cosets of the stabilizer group.

For an operator $F$ and group $\mathcal{G}$, the coset\footnote{Note that we will only use left cosets in this thesis.} $F\mathcal{G}$ is determined by $F$ is defined by
\begin{equation}
    F \mathcal{G} = \{F g \mid \ g \in \mathcal{G}\}.
\end{equation}


By defining $M =  L_1^\dag L_2$, we can rearrange (eq. \ref{eq:two_logical_operators_acting_identical_on_all_codeword_states}) to
\begin{equation}
    M \ket{\Phi} = +1\ket{\Phi}  ; \forall \ket{\Phi}\in \mathcal{L}.
\end{equation}

We see that the operator $M$ is by definition (eq. \ref{eq:defintion_codespace_whole_group}) an element of the stabilizer group, because all codewords are its $+1$ eigenstates.
If we rearrange the definition of $M$ we get the that $L_1$ and $L_2$ are in the same coset of the stabilizer group
\begin{equation}
    L_2 = L_1 M \text{ with } M \in \mathcal{S}.
\end{equation}
We can associate this coset with the logical operation $L_1$.

Thus all the different possible choices of implementation of the same logical operator are contained in the same coset of the stabilizer group.


All elements of the same coset of the stabilizer group act therefore the same on all codewords.
They all realize the same logical operation. 
We see that there is no 1-to-1 correspondence between logical operations and possible implementations but to a coset of operators.

The collection of cosets also from a group called the quotient group $\mathcal{C}(\mathcal{S})/\mathcal{S}$ we find that it is equal the group of all distinct logical operations \footnote{
    Note that we keep track of the phase. One can do the same analysis for the sets in which we disregard the global phases.
} on the encoded qubit and as such is isomorphic to the Pauli group $\mathcal{P}_k$ on $k$ qubits
\begin{equation}
    \mathcal{C}(\mathcal{S})/\mathcal{S} \cong \mathcal{P}_k.
    \label{eq:centralizer_stbailizer_quotient_group_logicals}
\end{equation}
We can therefore realize logical operations on the encoded state by using an operator of the corresponding coset of the stabilizer group in the centralizer.
The exact choice of element from the coset is not important for our use case\footnote{
    Note that the choice is rather important for experimentalists, due to the difference in weight of the operator and support on the physical qubits between the possible realizations 
    }, all elements are logical equivalent and they only differ by an element of the stabilizer group, 
which by stabilizer equivalence is to say, they do not differ.

The centralizer $\mathcal{C}(\mathcal{S})$ can therefore be partitioned into cosets of the stabilizer group, each realizing a unique logical operator.
We fix a set of operators, one for each logical operator from each cosets, and refer to the set as realizations of the logical operators $\bar{\mathcal{P}}$.

\end{equation}
where $\bar{P}\mathcal{S}$ is the notation for the right coset of the stabilizer group by applying the operator $\bar{P}$.

\subsubsection{Cosets of the Centralizer}\label{ssec:cosets_of_the_centralizer}

The centralizer can be partitioned into cosets of the stabilizer group.
Each of those cosets is associated with a logical operator on the codespace.
Complementary to this, in subsection \ref{ssec:syndrome} we showed that each syndrome is associated with a subspace and each of those subspaces is isomorphic to the codespace.

Changing the subspace, changes the associated syndrome.
Conversely the syndrome associated with a given subspace can be changed by redefining the stabilizer group.
We thus consider the elements of the same Pauli group $\mathcal{P}_n$ and partition them into sets according to their error syndrome.

This results in a structure where all operators with the same error syndrome are collected in cosets of the centralizer, labeled by the syndromes. 
Similar to the previous section where all operators associated to the trivial syndrome are collected in centralizer.
We can choose for each syndrome a representative error $E_{\sigma}$.
In the following we assume that we fix this representative error once for all syndromes.

The whole Pauli group $\mathcal{P}_n$ can then be partitioned into cosets of the centralizer of the stabilizer group, where we go over all possible $2^m$ syndromes
\begin{equation}
    \mathcal{P}_n = \bigcup_{\sigma \in \{0,1\}^m} E_{\sigma} \mathcal{C}(\mathcal{S})= \bigcup_{\sigma \in \{0,1\}^m} \bigcup_{\bar{P} \in\bar{\mathcal{P}}} E_{\sigma} \bar{P} \mathcal{S}.
\end{equation}

All $2^m$ cosets of the centralizer have the same internal group structure.
Each of those cosets of centralizer can be partitioned further into the cosets of the stabilizer group associated with a logical operator, as described in equation \ref{eq:centralizer_partitioned_into_cosets_of_stabilizer_group},
where each of those cosets also inherits the representative error $E_\sigma$ of the associated syndrome.
As a result we can partition the whole Pauli group into cosets of the stabilizer group.
This structure is illustrated in figure \ref{fig:stabilizer_formalism_cosets_of_centralizer}.

We can rewrite any operator from the Pauli group $P \in \mathcal{P}_n$ as a combinations of three operators
\begin{equation}
    P =  E_\sigma \bar{L} S \forall P \in \mathcal{P}_n; S \in \mathcal{S}, \bar{L} \in \bar{\mathcal{P}}.
    \label{eq:pauli_rewriting_stabilizer_formalism}
\end{equation}
Using one operator each from the stabilizer group $\mathcal{S}$, the set of realizations of the logical operators $\bar{\mathcal{P}}$ and the set of canonical representation errors associated with the syndromes, the associated operator is denoted by $E_\sigma$.
Here the overall phase is contained in the logical operator $\bar{L}$.
If we do not care about the phase we can just use the phase-normalized realizations of logical operators.

\subsection{Stepping through a QEC Cycle using stabilizer formalism}\label{ssec:stabilizer_summary_qec_cylce}
\textcolor{red}{Adapting this to use a numbered is on my todo list, for after the presentation.}

Having introduced the necessary details of the stabilizer codes and both their linear and group structure,
let us now put the this together.
And ask the question: How does this help us to detect and correct errors?

To see this, we will talk through one QEC cycle, which describes the process of detecting and correcting an error.

To be able to detect errors we start in a logical codeword state\footnote{
We will talk at in section \textcolor{red}{Put ref here} about the preparation of a logically encoded qubit. 
Note that we do not care which codeword state this is, as such we do not know the logical information encoded in those $k$ qubits.
Also note that the logical operators, by definition, all commute with the stabilizers and therefore we can construct the syndrome, without collapsing any possible superposition of codeword states or learning anything about the encoded state. 
} $\ket{\Phi}$.

If an error operator
% \footnote{
    % We still assume that our error operator is an element of the Pauli group $\mathcal{P}_n$.
% } 
$\tilde{E}$ acts on our codeword state $\ket{\Phi}$. 
We get the resulting faulty state $\tilde{E}\ket{\Phi}$.

The goal at the end of the cycle is to recover the original codeword state $\ket{\Phi}$.
To achieve this we need to find a way to inverse\footnote{
As discussed in section \ref{ssec:intro_pauli_group}, the square of any operator is the identity, in addition to some global phase which has no physical relevance.
Therefore we could recover our original codeword state if we apply the exact same error to our faulty state $\tilde{E}\ket{\Phi}$.
We would obtain a global phase which we could ignore $\tilde{E}\tilde{E}\ket{\Phi} = \alpha \ket{\Phi}$, with $\alpha=\pm 1$.
} the error $\tilde{E}$.
As such our goal during the QEC cycle is to identify the error operator $\tilde{E}$, to determine its adjoint.
We can then apply the adjoint error operator again to recover our original codeword state $\tilde{E}^\dag\tilde{E}\ket{\Phi} = \ket{\Phi}$.

We start the process of identifying the occurred error by measuring the syndrome of this state\footnote{
The exact circuit/approach of extracting the syndrome of a given state will be discussed in section \ref{ssec:syndrome_extraction}.
}, which will give us the error syndrome of the error operator $\sigma(\tilde{E})$. 
The extracted error syndrome points us directly to the coset of the centralizer which contains the occurred error $\tilde{E} \in E_{\sigma(\tilde{E})}\mathcal{C}(\mathcal{S})$.

We thus know that the error $\tilde{E}$ must be an element of the set $\tilde{E}\in E_{\sigma(\tilde{E})}\mathcal{C}(\mathcal{S})$.
We do not know yet from which coset of the stabilizer group in the cosets of the centralizer this error is from.

Each coset of the stabilizer in the centralizer realizes a logical operator on the encoded qubit.
Asking the question, which coset the error belong to is asking the question, which logical error $\bar{L}_{\tilde{E}} \in \bar{\mathcal{P}}$ does the error contain.
Here we adapt the rewriting introduced in equation \ref{eq:pauli_rewriting_stabilizer_formalism} to the occurred error 
\begin{equation}
    \tilde{E} = E_{\sigma(\tilde{E})}\bar{L}_{\tilde{E}}\tilde{S}.
\end{equation}

The problem of determining from which cosets of the stabilizer the error is part of, or which logical operator it includes, is the so called decoding problem.
It is handled by so called decoders\footnote{ 
We will go into more detail about possible approaches to decode a syndrome in the following section \ref{sec:decoding_problem_general}.
}.
For now lets assume our decoder give us a prediction from which coset of the stabilizer group the error operator should be from.
This prediction can be represented as a logical operator, which we refer to as $\bar{L}_D\in\bar{\mathcal{P}}$.

We can then construct a possible\footnote{
Note that we do not have any information about the stabilizer component $\tilde{S}$ of the occurred error, but as we will see later we can choose any stabilizer element $S_R$ for our recovery.
} recovery operation $R$ from the given parts as,
\begin{equation}
    R = E_{\sigma(\tilde{E})}\bar{L}_{D} S_R.
\end{equation}
where combined the representative error $E_{\sigma(\tilde{E})}$ obtained by the syndrome of the error, with the logical component $\bar{L}_D \in \bar{\mathcal{P}}$ predicted by the decoder.

By applying this recovery operation $R^\dag$, we hope to be able to undo the previously occurred error $E$, so we would succeed if  
\begin{equation}
    R^\dag E \ket{\Phi} = \ket{\Phi}.
\end{equation}

The final question is: 
Did we recover the state correctly?
So lets look at the state\footnote{
    Given that we can ignore global phase of a state we could also apply $R\tilde{E}$, instead of $R^\dag\tilde{E}$ and we would change the phase by commuting the operators to the correct position.
    But I found it more illustrative to recover the Pauli by applying the adjoint.
    } $R^\dag \tilde{E} \ket{\Phi}$,
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = S_R^\dag \bar{L}_D^\dag E_{\sigma(\tilde{E})}^\dag E_{\sigma(\tilde{E})}\bar{L}_{\tilde{E}}\tilde{S}\ket{\Phi}.
\end{equation}

Applying any Pauli operator together with its adjoint is identical to the identity, so $E_{\sigma(\tilde{E})}^\dag E_{\sigma(\tilde{E})} = I$, which results\footnote{
Here we follow the assumption that we fix the representative errors in the beginning.
} in 
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = S_R^\dag \bar{L}_D^\dag \bar{L}_{\tilde{E}} \tilde{S}\ket{\Phi}.
\end{equation}
The adjoint of the stabilizer group element $S_R$ commutes with the realizations of the logical operators, which all belong to the centralizer.
Therefore we can rewrite the expression as 
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = \bar{L}_D^\dag \bar{L}_{\tilde{E}} S_R^\dag \tilde{S}\ket{\Phi}.
\end{equation}
By definition, stabilizers acting on the codeword states do not change the state.
Resulting in
\begin{equation}
    R^\dag \tilde{E} \ket{\Phi} = \bar{L}_D^\dag \bar{L}_{\tilde{E}} \ket{\Phi}.
\end{equation}

We thus must be get the prediction of the logical 
\footnote{
Note that the choice of stabilizer element of the recovery $S_R$ does not matter, as such we only need to determine the correct cosets of the stabilizer group, not the identical error operator.
If they are not identical, they would implement a different logical operator $\bar{L}_\Delta = \bar{L}_D^\dag \bar{L}_{\tilde{E}}$, 
as indicated by the isomorphism between the chosen realizations of the logical operators $\bar{\mathcal{P}}$ and the Pauli group for the the encoded logical qubit $\mathcal{P}_k$.
Which then would change our codeword resulting in an unsuccessful recovery.
} component of the error correct $\bar{L}_D=\bar{L}_{\tilde{E}}$, to recover our original codeword state $\ket{\Phi}$.

We therefore see that the crucial part of the QEC cycle, and consequently its success rate, is dependent on solving the decoding problem correctly.
As such the choice of decoding strategy is quite influential on our QEC code performance.

Fortunately, there exist approaches that exploit properties and structure of the stabilizer codes to solve this task, utilizing mappings to mathematical optimization problems or to well-studied physics problems from other areas. 
We will investigate those approaches in the following section \ref{sec:decoding_problem_general} in detail.

We can quantify the decoding success by a logical error rate $p_L$, the rate at which we make logical mistakes in our identification\footnote{
    Given that we either assume that our recovery is perfect or that we do not actually apply a recovery by using Pauli frame tracking, see section \ref{sec:pauli_frame_tracking}.
} of the occurred error.

\section{Decoder}\label{sec:decoding_problem_general}

We refer to the classical algorithms solving the decoding problem as decoders.
They try to predict the occurred error, or the logical part of it, based on the observed syndrome. 

% To shortly recap, given the error syndrome we can identify the cosets of the centralizer to which the error belongs.
% For a successful recovery we also need to know to which coset of the stabilizer group, disregarding the phase, the error belongs to.
% The coset of the stabilizer group defines determined logical component of the occurred error.
% The syndrome does not provide any information\footnote{
%     Knowing the logical state before or after the error occurred would lead to a wavefunction collapse/a projection onto the measured state and as such would destroy possible superpositions of the logical state.

The syndrome only identifies the cosets of the centralizer containing the error. 
As discussed in the previous section, successful recovery additionally requires to correct prediction of the logical component of the error, which is determined by the coset of the stabilizer group the error belongs to.
To infer this information we rely on an error model as an additional resource.

The error model assigns a probability $\text{Pr}(E)$ to each error operator $E$, which can be used by the decoder to determine which error or set of errors has likely occurred. 
The error models used in this thesis are introduced in section \ref{ssec:error_models}.

We will use two different decoding strategies in this thesis.
The first is called \emph{Maximum Likelihood Decoding} (MLD). 
This decoding approach has the highest likelihood of returning a successful recovery.
It can be summarized, in the context of stabilizer codes, as determining the coset of the stabilizer group, in the coset of the centralizer with the corresponding syndrome, with the highest total probability of occurring.
As such we need to compare the total probability of each coset, for which we take into account all errors.
In return we are able to determine the recovery operation with the highest probability of a successful recovery.
We will go into more detail in the following section \ref{ssec:general_ML_decoder}. 

% For general stabilizer codes, solving the MLD problem has been shown to be a $\# P$-hard \cite{MP_sharp_P_hard}.

The second decoding approach considered in this thesis is called \emph{Minimum Weight Perfect Matching} (MWPM).
Compared to MLD, it generally achieves a lower probability of successful recovery but can generally be solved more efficiently. 
MWPM uses structures in the interaction between stabilizers and occurred errors to identify the most likely error event consistent with the observed syndrome.
We will go into more detail in the section \ref{ssec:general_MWMP_decoder}. 

The main difference between MLD and MWPM is that MLD determines the overall most likely recovery by identify the logical error with highest total probability, obtained by summing over all corresponding physical errors. 
In contrast MWPM identifies the single most likely physical error consistent with the measured syndrome.

The number of possible syndromes scales exponentially with the distance of the code for the surface code\footnote{This will be discussed in detail in section \ref{sec:kitaev_surface_code}.}.
As such it becomes unfeasible to compute a look up table ahead of time for larger code distances. 
But under certain conditions\footnote{We will discuss those conditions a bit when we talk about Pauli frame tracking \ref{sec:pauli_frame_tracking}.} we need to be able to correct the errors in real time or close to it \cite{terhal_qec_formalism}.
Therefore efficient decoding techniques are relevant to enable real time decoding and correction of occurred errors during QEC cycles. 


\subsection{Maximum Likelihood Decoding}\label{ssec:general_ML_decoder}

In this section we will formalize the \emph{Maximum Likelihood Decoding} (MLD) strategy for stabilizer codes \cite{mld_decoding_intro_luis}.
The MLD problem consists of identify the logical coset\footnote{We can and will once again disregard overall phase of this logical operator.} of the stabilizer group $\bar{L}_MLD\mathcal{S}$, in the centralizer with the corresponding syndrome, with the highest total probability of occurring, for the given error model.
We can formalize this as
\begin{equation}
    \bar{L}_{\text{MLD};\sigma} = \arg \max_{\bar{L} \in  {\bar{\mathcal{P}}}} \text{Pr}(E_\sigma \bar{L} \mathcal{S}), 
\end{equation}
where $\bar{L}_{\text{MLD}}$ is the prediction for the logical component of the error and
$\text{Pr}(E_\sigma \bar{L}\mathcal{S})$ is the total probability of the cosets of the stabilizer group $\mathcal{S}$. 
The representative error $E_\sigma$ fixes the syndrome the measured syndrome. 
We then maximize over the logical components $\bar{L} \in {\bar{\mathcal{P}}}$, taking all cosets in the centralizer into account.

We can expend this equation in terms of probabilities of individuals Pauli operators as 
\begin{eqnarray}
    \bar{L}_{\text{MLD};\sigma} = \arg \max_{\bar{L} \in  {\bar{\mathcal{P}}}} \sum_{S \in \mathcal{S}} \text{Pr}(E\bar{L}S). 
\end{eqnarray}
The number of operators in this summation grows exponentially with the distance of the codes as the number of stabilizer elements of the stabilizer group grows exponentially\footnote{
    This will be shown in the section introducing the surface code (section \ref{sec:kitaev_surface_code}) where we discuss in detail how the number of generators of the stabilizer group scale with code distance. 
} with the distance of the code.

Note that by taking all possible operators into account the ML decoder takes the degeneracy of errors into account.
As such the MLD finds the coset with the highest combined probability of all errors, not only the error with the highest probability.
Therefore even if we have a single physical error in a different coset that has the highest probability of occurring, MLD will find the coset with the overall highest probability.
This is a crucial difference to the following MWPM decoder, as described in section \ref{ssec:general_MWMP_decoder}.

The problem of determining the correct logical component using MLD has been shown to be $\#$P-hard for arbitrary stabilizer codes \cite{MP_sharp_P_hard}.

% make the following more precise, which models and why?
The MLD problem of stabilizer codes can be mapped to the partition function of classical disordered spin models \cite{topological_quantum_memory}. 
For this the summation over different Pauli operators is mapped to the partition function of a classical disordered spin model.
Dependent on the specific model and its symmetries, some of those classical disordered spin models can be solved exactly \cite{topological_quantum_memory,fundamental_thresholds}.
For those we can identify a phase change in the model at the threshold probability, which we will discuss in section \ref{ssec:threshold_theorem}. 
For some combinations of QEC and error model the MLD problem has been shown to be efficiently solvable \cite{efficient_decoding_algorithm}.

If we can solve the MLD problem exactly, we can calculate the success rate of the MLD $p_{\text{success}}$ exactly, 
by summing over the probabilities of the most likely cosets 
\begin{equation}
    p_{\text{success}} = \sum_{\sigma \in \{0,1\}^m} \text{Pr}(\bar{L}_{\text{MLD};\sigma}\mathcal{S}).
\end{equation}

\subsection{Minimum Weight Perfect Matching}\label{ssec:general_MWMP_decoder}

The second decoding strategy is called Minimum Weight Perfect Matching (MWPM)\cite{pymatching,pymatching_loger_paper}.
We can summarize the goal of MWPM as finding the physical error operator with the highest probability of occurring consistent with the observed syndrome.
To highlight the downside of this approach as we determine a single operator, we are not taking the degeneracy of possible errors into account or 
even the possibility that different logical classes/cosets could have multiple errors with the lowest weight\footnote{
    Note as well that there exist approaches where the degeneracy is also included in such maximum probability decoders \cite{degeneracy_enhanced_mp_decoding}.
}.

The MWPM decoder is based on mapping the error model with the stabilizers onto a Detector Error Model (DEM) which we then map onto a weighted graph, called the matching graph.

To get from the error model to the DEM, we determine how each error mechanism changes the bits of the syndrome.
We represent each bit of the syndrome as a detector and if the bit is changed by an error we say that the error triggered the corresponding detector.
We can then use the connection between physical error mechanism and those detectors, by associating each error mechanism with bit of the syndrome they flip.
This correspondence to identifying with which generator of the stabilizer group the error mechanism anticommutes.
The DEM contains the mapping which error mechanism trigger which detectors.


To keep tack of logical changes we label each error in the DEM with the additional information on how the associated error mechanism commutes with the logical observable.
For this we first need to fix a certain realization of logical operator, which is done by fixing the set $\bar{\mathcal{P}}$. 
We then label the each error mechanism that anticommutes with the logical observable.
This labeling will be used to determine the effect of the determined most probable error on the chosen implementation logical observable. 
We therefore can predict how the measurement of chosen implementation of the logical observable would change.
If the final error includes an odd number of anticommutating error mechanisms the most probable error will itself anticommute with the logical operator, changing its value.
For an even number we do not expect the measurement of the chosen implementation\footnote{
    Note here that even if for one choice of the logical operator, the measurement does not change. 
    A different implementation of the same logical operator could anticommute with the predicted error and therefore result in a flipped measurement. 
} logical operator to change. 

If each error mechanism triggers at most\footnote{For the surface codes errors on the edge only trigger one detector. To obtain an graphlike error model we can introduce a boundary node into our graph \cite{pymatching_loger_paper}.} two detectors, we can refer to the DEM as a graphlike error model.
For this to be a valid assumption we might need to decompose the $Y$-type errors into $X$-type errors and $Z$-type errors ($Y = i XZ$), to ensure the graphlike property of our error model.
We will at this point assume\footnote{This is a valid assumption for the surface code \cite{topological_quantum_memory}.} that we have such an error model.
We then solve the $X$-type and $Y$-type MWPM decoding problems independently.

We can map this graph like error model onto a matching graph by representing each detector as a node and each error mechanism as an edge.
Each edge represents an\footnote{Or multiple errors if all of them have the same effect.} error $E$ and is weighted $w(E)$ by the probability of the error $\text{Pr}(E)$ with
\begin{equation}
    w(E) = \log \left( \frac{1-\text{Pr}(E)}{\text{Pr}(E)}\right).
\end{equation}
Each edge connects the nodes, which represent the detectors that the associated error mechanism triggers. 

The MWPM decoder then identifies the most probable error event consistent with the syndrome.
Finding a error consistent with the syndrome correspondence to finding a set of edges, such that all triggered detectors have an odd number of incident edges and non triggered detectors have an even number of incident matches.
To choose the most probable of such configurations we need to find the set of edges, that fulfills the above condition with the lowest total weight $\sum_E w(E)$. 
Note at this point that we are following here the definition of an minimum weight embedded matching, which is a variation of the MWPM problem. 
It is the basis of PyMatching the MWPM decoder which we are going to use \cite{pymatching_loger_paper}. 

After finding such a set the decoder then returns the prediction if the logical observable commutes or anticommutes with the predicted error, by combining the labels on the predicted edges, as previously described. 
As such it predicts the change of the logical observable due to the error.

We could also use the configuration of edges to reconstruct the predicted occurred error.
But as we are only performing a memory experiment with only Clifford operation we can postpone the decoding to after the circuit is done executing and then reinterpret the results, as discussed in the section \ref{sec:pauli_frame_tracking} discussing Pauli frame tracking. 
The above explained process already included the commutation of the predicted error with the observed measurement and therefore directly predicts the expected measurement.

As we discussed in section \ref{ssec:syndrome} each subspace associated to a syndrome is isomorphic to the codespace.

\section{Surface Code}\label{sec:kitaev_surface_code}

The specific QEC code from the class of the stabilizer codes we are using is the surface\footnote{We use the unrotated version because an efficient MLD decoder exist for this QEC code \cite{efficient_decoding_algorithm}.} code \cite{kitaev_surface_code,other_surface_code_paper}, which is can be derived from the toric code \cite{kitaev_toric_code}.
The difference between the toric code and the surface code is the change from periodic boundary conditions of the toric code to open boundaries for the surface code.
The above papers and the paper by Dennis \cite{topological_quantum_memory} are the references of this chapter.

The surface code is a well established QEC code, that is used due to it advantageous properties.
One of those advantageous properties is that the surface code can\footnote{We can and will use a syndrome extraction circuit that requires more than NN interaction and therefore this is only partly valid.} be constructed using only next neighbor interactions.

To visualize the toric code of distance $d$, we arrange $n=d^2 + (d-1)^2$ physical qubits on a $d \times d$ lattice,
by placing the qubits on the nodes connecting the sites of the lattice.
Note that for the surface code with open boundary conditions we additionally remove the notes connecting the upper most lattice sites to form a rough boundary.
The resulting constructing can be seen in figure \ref{fig:surface_code_explaination} a), with the associated indexing we are using for the qubits.

As described in section \ref{sec:stabilizer_formalism} to specify a QEC code of the class of stabilizer codes, we define the stabilizer group of the code.

This set can be split up into site and plaquette stabilizers\footnote{

We can formalize site stabilizers as 
\begin{equation}
    S_{X,i} =  \bigotimes_{j \in A_i} X_j,
\end{equation}
where $A_i$ is the set of physical qubits that sit on the edges incident to the $i$-th stabilizer site.
We can express the plaquette stabilizers as 
\begin{equation}
    S_{Z,i} =  \bigotimes_{j \in B_i} Z_j,
\end{equation}
where $B_i$ are the edges forming the cell of the $i$-th plaquette stabilizer.

Note that the site and plaquette stabilizers in the bulk act on $4$ qubits each, whereas the generators on the edge only act on $3$ qubits each.

We can easily check that all generators commute because site and plaquette stabilizers always overlap on an even number of qubits, either $0$ or $2$. 
And site and plaquette stabilizers trivially commute with other stabilizers of their type.

\section{Fault Tolerance and Error Models}\label{sec:fault_tolerance}

In the previous section we implicitly assumed that errors only happen at a specific point in our quantum circuit.
Right after we encoded our state and then they stop occurring when we want to extract the syndrome and recover the state.
This is assumption is not well motivated as in realistic implementations, error can occur in every part of the circuit and with every gate applied.
Therefore we need to change our error model.
We will talk about the specific error models we will look at in this thesis in section \ref{ssec:error_models}, and the one that is described above is called the circuit level error model.

At this point we introduce the notion of fault tolerance in relation to QEC codes.
The concept of fault tolerance describes how our QEC code behaves under the condition that faults happen during its execution.
Given that all components of a circuit can fail, we can not expect of our QEC code to recover the original state.
For fault tolerance we change the requirements.
If we previously had a distance $d$ QEC circuit, that was able to correct $t$ errors, we now require of our fault tolerant circuit distance $d$ circuit, that $t$ faults lead to at most $t$ errors at the end of the QEC cycle.
As such the resulting state should be a correctable one with the same circuit given the noiseless assumption \cite{gottesmann_book}.
We refer as such a QEC as distance preserving and we will refer to the error remaining after the QEC cycle as the residual error.

One reason $t$ faults could lead to more than $t$ faults at the end of a circuit is so called error propagation, which we will talk about using the example of a CNOT gate.

\subsection{CNOT Gate}\label{ssec:cnot_gate}
As an example for error propagation and because we will use its properties later we will shortly discuss the CNOT gate.

It is a two qubit gate and implements the controlled NOT operation.
At especially important for the use case in this paper we look at how propagates errors $E$ \cite{gottesmann_book}.
As the CNOT operation is unitary and hermitian we can write
\begin{equation}
    \text{CNOT} \ E\ket{\Psi} = \text{CNOT } E\text{ CNOT} \ \text{CNOT} \ket{\Psi}.
\end{equation}
The propagated error is therefore $\text{CNOT } E \text{ CNOT}$.
We can identify that an error $E = X \otimes I$, will be propagated to become $X \otimes X$.
Here the first element of the tensor product is the error acting on the control qubit and the second is acting on the target qubit of the CNOT gate. 
As such the CNOT gate propagates $X$-type errors from the control bit onto the target bit. 
% Show a circuit diagram for CNOT gates!

We can identify such a propagation also in the other direction.
$E = I \otimes Z$, will become $Z \otimes Z$, which tells us that $Z$-type errors propagate from the target qubit onto the control qubit.

As such even if the CNOT gate itself is not faulty, if we an error is propagated through a CNOT gate, the number of errors in the circuit is increased.
And the CNOT gate propagates errors in both directions from control onto the target and vice versa.

We will later show that we can use this error propagation property of CNOT gates to our advantage in section \textcolor{red}{Put ref to syndrome extraciton here}.

\subsection{Threshold Theorem}\label{ssec:threshold_theorem}

The cornerstone of QEC codes and there usefulness is based on the threshold theorem, which states that even under the assumption that all components are noisy we can still arbitrary reduce the logical error rate of the logical qubit, given that the error rates of the noisy components is below a certain threshold \cite{gottesmann_book},\cite{topological_quantum_memory}.
As such we can if we the physical error rate is below the threshold we can reduce the logical error rate by using more resources.
This can be done for surface codes by increasing the distance $d$ of the code.
The overhead in the number of qubits has been shown to be polynomial \cite{topological_quantum_memory}.

Based on this theorem we not need to realize perfect qubits in an experimental setting to be able to realize arbitrarily long quantum computations
Instead we can use QEC codes to suppress the logical error rates to levels where long quantum computations become feasible.  
This makes quantum computational possible as awe do not need to create perfect qubits, just enough decent qubits, to enable quantum computing.

Note that will below the threshold each additional qubit reduces the logical error probability, above the threshold the logical probability actually rises for each qubit added.

The threshold is QEC and error model dependent.
The threshold is an important quantity of a QEC codes. 
Codes with higher threshold can be used with less optimal qubits and therefore might enable quantum computation where other QEC codes still fail. 

The threshold can be determined through mapping to statistical physics models, as alluded to in section \ref{sec:ML_decoding}. 
In those models we can identify a phase change between an ordered and unordered state at the threshold \cite{fundamental_thresholds},\cite{topological_quantum_memory}.
 
We can also determine the threshold using numerical simulations of the QEC code with an associated error model.
This will be the approached used in this thesis.
To determine the threshold we compute the logical error rate for different distances and different physical error rates. 
Above the threshold the higher distance codes should have a higher logical error rate, whereas there increased distance results in a lower logical error rate below the threshold.
Given that we want to determine the location of a phase transition, when we want to locate the threshold, we can use finite-size scaling analysis to determine the value of the threshold give the data.

\subsubsection{Circuit Level Noise Model}\label{sssec:circuit_level_noise}

The other noise model we will use is the so called circuit level noise model.

Starting with incoming errors, which are the errors already present on the data qubit before we start our syndrome extraction circuit.
These would be the only errors for a code capacity error model and the circuit is by definition built to detect those.
Given that we look at the circuit which extract the plaquette stabilizer syndromes, the incoming $X$-type errors are propagated to the ancilla qubits, while $Z$-type errors commute with the control side of the CNOT gate.
As a result we can detect each of those errors individual and the circuit behaves as expected.

Looking at possible errors on the initialization of the ancilla qubit, we can determine that a possible $Z$-error would be propagated onto each data qubit and therefore realize the stabilizer we want to measure.
Due to stabilizer equivalence we can ignore this error.
A possible $X$-type error would not be propagated onto the data qubits, but it would change the result of the measurement, and is therefore in principle a measurement error.
We will discuss the influence of measurement errors at the end.

The next possible fault location in a circuit level noise model are after each CNOT gate.
Here each possible $X$-type error on the ancilla qubit is in principle a measurement error.
For $Z$-type errors only those that can propagate onto data qubits are relevant.
$Z$-type errors after the last CNOT can be neglected, as they do neither propagate to the data qubits nor affect the measurement outcome.
Errors after the first and third CNOT are equivalent, up to a stabilizer, to a weight one error $Z$-error on the data qubits and therefore do not reduce the distance.
Here we used stabilizer equivalence 
\begin{equation}
    I \otimes Z_2 \otimes Z_3 \otimes Z_4 = Z_1 \cdot S_Z,
\end{equation} 
to rewrite a weight-three error as an equivalent weight-one error and a stabilizer. 
In contrast, a $Z$-type error after the second CNOT gate will propagate through the following two CNOT gates and will produce a correlated weight-2 error on the data qubits.
Such an error is called a hook error \cite{topological_quantum_memory}.
By choosing the CNOT ordering such that the resulting hook errors do not align with the logical operator, we can assure that these correlated errors to not result in a lower weight logical operator.
It still takes $t + 1$ independent errors events to realize a logical operator, and the distance of our code ist preserved.

Lastly we take a look at the effect of measurement errors.
A single measurement error leads two to detection events separated in time, a so called time like error. 
As an example take an circuit without errors on the data qubit only with two measurement error, sharing the type of stabilizer they act on.
Such two measurement errors are indistinguishable from the syndrome of a chain of data qubits error connecting those to stabilizer.
If our decoder then assumes that the chain of data qubits is the error that has occurred, the correction would be the same chain again. 
By applying the correction we would introduce a chain of errors connecting those two stabilizer onto the previous error free stabilizer.
As such we might have introduced an error of higher weight than two onto our logical error with just two errors.
This could possibly introduce a logical operator in cases where the data qubits are not free of errors to begin with.

To overcome this problem we need to repeat the syndrome extraction, to construct a syndrome history, which we then can decode.
Measurement errors then appear as error separated by time instead of by space and we can decode this three dimensional space time matching problem, using the MWPM decoder.
Note that for a finite time interval measurement errors in the last syndrome extraction step, are never distinguishable from spatial errors \cite{topological_quantum_memory}. 
A possible solution for this is the overlapping recovery method introduced by Dennis et al. in \cite{topological_quantum_memory}.

As such the time direction has the same dimension as the space directions.

We will use an approximation in this thesis, by giving the decoder access to a final noiseless syndrome extraction circuit.
This in combination with a noiseless syndrome to start with, gives the decoder all necessary resources to be able to distinguish time like errors from space like errors.
Note that this is an strong assumption leading to a better performing code, but we can use this as an upper bound for the possible performance capabilities of this standard syndrome extraction circuit. 
 
We can summarize the resources needed for each QEC cycle, in which we perform $d$ rounds of syndrome extraction.

If this is not the case the number of required qubits is $n_{\text{ancilla}}'= d \cdot n_{\text{ancilla}}$. 
Then we are interested in the number of measurements performed $n_{\text{meas.}}$, the time steps required $n_{\text{time}}$, assuming we can perform CNOTs on a disjoint set of qubits in parallel.

,where $m$ is the number of generators of the stabilizer group and $d$ is the distance of the code.

\subsubsection{Stean Type Error Correction}\label{sssec:steane_type_error_correction_intro}

The main topic of this thesis is determining the threshold of Steane Type Error Correction (STEC) and to compare it with the SSE.
As such we need do introduce STEC, which is named after Steane \cite{steane_type_EC_og_paper}.

As STEC can be used as a syndrome extraction circuit for CSS codes \cite{gottesmann_book}.
It is based the fact that for all CSS codes the logical CNOT ($\bar{\text{CNOT}}$) is transversal.
We can then perform a logical trivial circuit while using the properties of CNOTs gates, as introduced in section \ref{ssec:cnot_gate}, to propagate the error onto ancilla qubits.

The main difference is that we require two verified logical encoded qubits as a resource and we need a platform which enables more than next neighbor interaction\footnote{Such as the platform used in \cite{friederike_steane}.}.
For this additional cost we get a syndrome extraction circuit which only takes a constant amount of time and therefore also limit the interaction of the ancilla qubits with the data qubits.

To formalize the logic behind STEC, we introduce the notation $\ket{\Phi}_L$ is the logical codeword state, and $\ket{+}_L = 1/\sqrt{2} (\ket{0}_L + \ket{1})$ is the logical encoded ancilla state \cite{friederike_steane}.
We can then use the logical CNOT gate to perform the operation
\begin{equation}
    \text{C}^{\text{data}}\text{NOT}^{\text{anc.}}(\ket{\Phi}_L \otimes \ket{+}_L) = \ket{\Phi}_L \otimes \ket{+}_L.
\end{equation}
In the error free case this is a trivial operation on both logical qubits, as the ancilla $\ket{+}_L$ qubit is in an eigenstate of the logical $\bar{X}$ operation.

But for cases where there is a $X$-type errors on the $i$th data qubit, written as $X_i$, the error will get propagate through the CNOTs acting between the individual logical and ancilla qubit.
The error will end up in the same location on the ancilla qubit
\begin{equation}
    \text{C}^{\text{data}}\text{NOT}^{\text{anc.}} (X_i^{\text{data}} \otimes I^{\text{anc.}} ) (\ket{\Phi}_L \otimes \ket{+}_L) = X_i^{\text{data}} \ket{\Phi}_L \otimes X_i^{\text{anc.}}\ket{+}_L.
\end{equation}
We can then construct the measurements of $Z$-type stabilizer generators by measuring each qubit in the $Z$-basis and recombining the measurement results to construct the measurement of the generators of the stabilizer group.
Note that we can also reconstruct the logical measurement of the $\bar{Z}$ of the ancilla, but the result will be random and will not give us any information about the state of the encoded data qubit.

While we use the propagation of errors from the data qubit onto the ancilla qubit, the propagation in the other direction still contributes to a residual error.
The whole process is fault tolerant by construction, since the logical CNOT is transversal and therefore can only propagate at most one error onto each encoded code block.
Moreover each measurement error will behave like a single data qubit, and as such is also fault tolerant. 
This is because we measure each qubit at once and afterwards recombine, we can observe that each measurement error correspond to a single error on a data qubit, leading to two flipped syndrome bits on the corresponding adjacent stabilizers\footnote{In the case of the bulk, but this is enough to clarify this point.}.
Note here the difference to the previous method, where a single measurement error only flipped a single syndrome bit.

The decoding problem with those syndromes than corresponds to the task of decoding the error on the ancilla qubit, including the measurement errors. 
The difference between the data qubit and the ancilla qubit will be accepted as part of residual error, as it at most introduces an error on the data qubit for each occurred error and is therefore still fault tolerant.
Therefore a single round of syndrome extraction suffices to enable a fault tolerant QEC cycle.

The method described above is able to determine the $Z$-type part of the syndrome which detects $X$-errors.
Due to symmetry the detection circuit for $Z$-type errors, so the measurement of the $X$-type stabilizer generators works nearly identical.

We only change the ancilla to $\ket{0}_L$ and switch the orientation of the logical CNOT gate, using the ancilla as control qubit and having the data qubit as a target.
Therefore $Z$-type errors will be propagated onto the ancilla where they will be measured by the parity checks and $Z$-type errors will contribute to the residual error.

We will refer to each of those constructions as half-cycles.
Note that we can influence the composition of the residual error by the choice of order of those half-cycles.
The type of error which we will correct first will occur more often in the residual error for circuit level noise. 
This is because there are more possible locations for errors that are not contributing to the corresponding syndrome part.
We will show this again in \textcolor{red}{Put ref to propagated error model here.}.

We can use this asymmetry in the order also to our advantage. 
The error that is correct first has the advantage that less are added through circuit level noise by the syndrome extraction circuit, as such it might be able to recover from some errors that otherwise might obtain to much weight when their syndrome is extracted later.
Therefore the ordering of the circuit can be used to optimize if whatever is placed between QEC cycles is more susceptible to one kind of error or produces one kind of error more often. 

Note that in our case for a memory experiment with multiple rounds the ordering does not matter, as we effectively get a symmetric bulk after the first and before the last correction.
As long as we want to correct\footnote{Or in our case if we want to pretend to correct both type of errors.} both type of errors there is no advantage in changing the order from an alternating ordering for a memory experiment.

We can quantify the resources needed for the cases where we want to use STEC to extract the syndrome for a logical qubit encoded in the surface code of distance $d$ with
\begin{align}
    n_{\text{ancilla}} &= 2n = 2 (d^2 + (d-1)^2) =  4d^2 -4d +2 \\
    n_{\text{meas.}} &= 2n  = 4d^2 -4d +2  \\
    n_{\text{time}} &= 2  \\
    n_{\text{CNOT}} &= 2n  =   4d^2 -4d +2\\
\end{align}
Note that we only take the time into account where the data qubits are needed, therefore we ignore the time steps needed for the measurement of the ancilla qubits and the time needed to encode the ancilla qubits.
The basis for this assumption is that we do not block the logical qubit, and the preparation and measurement can be done in parallel, to continued operations on the qubit.
But this is also only a minor detail, the core advantage is that the time needed is constant and therefore independent of the distance of the code.
Note as well that for those calculations of resources we neglected the resources needed for the encoding of the logical ancilla states.

The whole approach of STEC can be seen as a way to reduce the problem of syndrome extraction to a syndrome of state synthesis \cite{steane_type_EC_og_paper}. 
As such the exact choice of a preparation circuit will influence the performance of the STEC approach. 
But for a functioning QEC code we anyways have to find a way to fault tolerantly and verifiable create an basis state.
Therefore this problem is not only required by STEC. 

Note that we require that we know the syndrome of the used ancilla state with certainty.
Otherwise we face the same problem described in the previous chapter, in uncertain syndrome errors lead to the possible implementation of high weight errors.
This would also stop the STEC from working properly.
For this thesis we assume that we have access to such ancilla states, with a determined syndrome and independent errors on each data qubit of the ancilla. 

To summarize we can highlight that STEC is an approach to syndrome extraction which enables constant in time syndrome readout, no matter the distance of the code, at the cost of preparing a more complex ancilla state. 
In the process STEC minimizes the coupling between the data registry and the ancilla qubits and therefore reduces the noise obtained from the ancillas. 
The costs is that we have to find a way to prepare fault tolerant logical encoded qubits with a trusted syndrome, a task which need to solve anyways to obtain a completely functioning QEC code.

\section{Pauli Frame Tracking}\label{sec:pauli_frame_tracking}

The last step of a QEC cycle is the recovery and in this section we discuss a way to implement this recovery without applying any operation onto the state.

The process we are going to use is called Pauli Frame Tracking \cite{pauli_frame_tracking_knill}.
The basic idea follows is based on concept that we already introduced surround the introduction of equation \ref{eq:pauli_rewriting_stabilizer_formalism}.
We can treat the recovery the same way as the errors, splitting it up into a canonical representation of the syndrome $E_\sigma$, determined through the extracted syndrome, and a logical component $\bar{L}$ given by the decoder.
This combination is the so-called Pauli frame.

As we have discussed already in section \ref{ssec:syndrome}, each syndrome can be associated with an subspace which is isomorphic to the codespace.
It is therefore not really necessary to apply the canonical representation of the syndrome $E_\sigma$ as a correction to the circuit, to map the state back to the $+1$ eigenspace of all generators of the Pauli group. 
We can also redefine our codespace to be the orthogonal space in which the state is now placed, which is isomorphic to the codespace.
This can be done by taking the logical XOR of the new error syndrome with syndrome of the redefined codespace in classical post processing.
Or by redefining the generators of the stabilizer group, switching the signs of the generators to redefine the trivial syndrome to point to the codespace. 

Under the assumption that the circuit consists of only Clifford gates\footnote{
Note that to be able to perform general quantum computations we need to be able to also perform non-Clifford operations. 
In those cases we need to be able to apply the correction in time \cite{terhal_qec_formalism}. 
But because we are only interested in a quantum memory simulation, we do not face this problem. 
}, we can also postpone the logical correction to a reinterpretation of the measurements.
The basis for this is that we can commute the Pauli frame through the complete circuit, as Clifford gates map Paulis to Paulis.
The Pauli frame is therefore only needed in the post processing after the execution of the circuit, and not while executing the circuit for our case.
